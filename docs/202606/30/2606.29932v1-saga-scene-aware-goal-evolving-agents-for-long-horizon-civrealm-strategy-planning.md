## 摘要
在复杂策略游戏中的长时域战略规划，需要在信息不完全和奖励稀疏的条件下，跨多个决策领域进行并发推理。现有的基于LLM的智能体存在三类系统性缺陷：由原始瓦片坐标导致的场景盲视、由整体状态转储导致的上下文溢出和领域耦合，以及将每个回合孤立对待的浅层跨游戏学习。我们提出SAGA，一个LLM多智能体框架，包含三种机制，每种直接针对一类缺陷：(i) 地图语义场景图，将游戏实体间的类型化空间关系编码到每个单位的自然语言上下文中，在不增加全局token的情况下解决空间盲视；(ii) 工具增强规划器，按需拉取细粒度领域状态，并将每个领域的指令分派给专用的专家控制器，消除上下文溢出、领域耦合和机械约束违反；(iii) 双时域反馈循环，结合周期性的游戏内目标生成和结构化的跨游戏因果事后分析，实现无需手动奖励工程的原则性战略演化。在FreeCiv上的评估表明，SAGA获得了最高的平均文明分数——环境中唯一的稀疏目标奖励，且方差低于两个最强基线，并且在基础设施建设（多目标冲突中最容易被牺牲的资源轴）上，它是唯一显著超越所有基线的方法。在大多数直接对局中，SAGA的得分超过两个最强基线，同时输出token（主导解码成本）减少了27%。配备跨游戏演化模块后，SAGA在连续五个回合中达到了链末最高分。消融研究证实，每个架构组件都独立地为这一优势做出了贡献。

## Abstract
Long-horizon strategic planning in complex strategy games demands concurrent reasoning across multiple decision domains under imperfect information and sparse reward. Existing LLM-based agents suffer from three systematic failures: scene blindness from raw tile coordinates, context overflow and domain coupling from monolithic state dumps, and shallow cross-game learning that treats each episode in isolation. We present SAGA, an LLM multi-agent framework with three mechanisms each directly targeting one class of failure: (i) a Map-Semantic Scene Graph that encodes typed spatial relations among game entities into per-unit natural-language context, resolving spatial blindness without global token inflation; (ii) a Tool-Augmented Planner that pulls fine-grained domain state on demand and dispatches per-domain directives to dedicated specialist controllers, eliminating context overflow, domain coupling, and mechanical constraint violations; and (iii) a Dual-Horizon Feedback Loop that combines periodic within-game goal generation with structured cross-game causal post-mortem, enabling principled strategic evolution without manual reward engineering. Evaluated on FreeCiv, SAGA attains the highest mean civilization score -- the environment's sole sparse objective reward -- with lower variance than the two strongest baselines, and is the only method that significantly surpasses every baseline on infrastructure construction, the resource axis most readily sacrificed under multi-objective conflict. It outscores the two strongest baselines in most head-to-head games while cutting output tokens (the dominant decoding cost) by 27%. Equipped with the cross-game evolution module, SAGA reaches the highest end-of-chain score across five successive episodes. Ablation studies confirm that each architectural component contributes independently to this advantage.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：在长时域、多领域的复杂策略游戏（如 FreeCiv）中，基于 LLM 的智能体面临三大系统性缺陷：
  - **场景盲视**：原始瓦片坐标缺乏关系语义，智能体无法理解空间上下文。
  - **上下文溢出与领域耦合**：整体状态转储随帝国规模增长至超 15k token/步，且单一输出将技术、军事、经济等决策混在一起，导致高优先级威胁挤占关键基础设施投资；高层规划常违反不可逆的机械规则，产生静默失败。
  - **浅层跨游戏学习**：现有方法（如 EpicStar）仅机械重放历史，缺乏结构化因果归因，无法实现原则性战略演化。
- **整体含义**：这些缺陷阻碍了 LLM 在类似《文明》系列这样需要长期信用分配、多目标权衡、稀疏奖励的复杂环境中的有效应用。SAGA 旨在通过创新的信息架构和反馈机制同时解决以上所有问题。

## 2. 论文提出的方法论：核心思想、关键技术细节

SAGA 是一个三机制 LLM 多智能体框架，对应三类失败模式：

- **地图语义场景图 (Map-Semantic Scene Graph)**：
  - 每回合构建图 \( G=(V,E) \)，节点为友方/敌方城市和单位。
  - 边分为友方-友方（协作导航）和友方-敌方（进攻目标/威胁告警）。标注曼哈顿距离（三级：close<3, medium<10, far≥10）和八方向。
  - 为每个实体生成简洁的自然语言上下文，例如：“Enemy Warriors (ID:52) medium (5 tiles) to the NorthEast.” 无需全局坐标，避免 token 膨胀。

- **工具增强规划器 (Tool-Augmented Planner)**：
  - 中心 Planner 是一个 ReActAgent，拥有三个按需工具：`query_city_metrics`, `query_army_roster`, `query_tech_status`。
  - 每回合只接收精简战略摘要，细粒度领域信息通过工具调用按需获取，状态大小自然扩展。
  - 输出结构化计划 `PlannerResponse`，将动作空间分解为五个并发域：city, civilian, military, tech, gov，外加串行化的 di plomacy。通过明确领域划分防止耦合。

- **双时域反馈循环 (Dual-Horizon Feedback Loop)**：
  - **内循环（游戏内）**：每 \( K=6 \) 回合执行一次。`AbstractAgent` 压缩动作日志为语义摘要，`SetGoalAgent` 生成时间有界、领域具体的子目标（如“10回合内生产 3 个 Settler 和 2 个 Granary”）。另设紧急重规划（新敌人出现或城市丢失）覆盖周期性边界。
  - **外循环（跨游戏）**：游戏结束后 `GameAnalyst` 进行结构化事后分析，识别根因并写入 JSON 报告。`PreGameStrategist` 在下一局开始前将其蒸馏为 `strategic_evolution_chain.md`，注入 Planner 的系统提示，实现跨游戏战略进化。

- **输出 token 降低 27%**（vs. 最强基线 CoS/HIMA），表明更精确的决策而非冗长计划。
- **跨游戏演化**：配备演化模块后，SAGA 在 5 轮连续游戏中达到最高终局分数（90），且所有基线中 SAGA 的轨迹最稳定（其他基线虽有提升但振荡剧烈）。

## 7. 优点

- **方法创新**：
  - 场景图提供了优雅的空间关系编码，避免全局坐标注入，在不增加 token 前提下实现场景感知。
  - 工具按需获取与解耦执行架构从根本上解决了上下文溢出和领域干涉，且强制执行机械约束。
  - 双时域反馈循环将稀疏奖励转化为密集子目标，并通过结构化因果分析实现原则性跨游戏进化，无需手工奖励工程设计。
- **实验设计亮点**：
  - 所有基线在完全相同的基础设施上重新实现，并使用相同 LLM 骨干，确保公平比较。
  - 统计方法全面：报告均值/中位数/四分位数，进行 Mann-Whitney 检验、多重假设校正、胜率分析，避免仅依赖均值。

- **结果可解释性**：详细分析了各组件贡献，特别是 Over-Militarization 问题被归因于缺乏内循环和外循环修复，揭示了架构设计的因果逻辑。

## 8. 不足与局限

- **输入令牌冗余**：层级设计导致 `SetGoalAgent`、`PlannerAgent`、`AbstractAgent` 接收重叠上下文，增加了输入成本。作者提到可通过共享内存或信息路由来优化，但未在当前工作中实现。
- **环境随机性未完全消除**：即使固定地图种子，野蛮人和海盗的生成时间/位置仍随机，导致军事指标的高方差（std > 40），可能掩盖一些真实差距。
- **单局过校正缺陷**：150 回合限制使单局内很难完美平衡所有目标，常出现过度军事化（如核心城市重复生产单位而忽略基础设施）。虽然外循环设计用于修复，但在弱骨干下修复效果有限。
- **弱骨干天花板**：当使用 GPT-4o-mini 时，即使正确的蒸馏战略也只能带来有限提升，因为执行层无法可靠地遵循复杂指令。这意味着 SAGA 的优势在骨干非常弱时会缩小。

- **未探索更长的游戏时长或更多对手**：实验限制在 150 回合和 2 个 AI，在更大规模和更长周期下表现未知。
- **未与强化学习方法对比**：论文明确声明不对 π 进行强化学习，也未与 RL-based agent 比较，因此非最优性声明。
