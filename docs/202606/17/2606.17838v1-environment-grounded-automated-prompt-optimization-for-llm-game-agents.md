## 摘要
交互环境中的LLM智能体对其提示高度敏感，但提示工程仍是一个手动的、特定于任务的过程。我们提出了一种面向LLM智能体的自动提示优化框架，该框架将观测到动作的流程分解为目标条件描述智能体和动作选择智能体，并通过环境奖励引导的LLM驱动进化循环迭代优化每个模块的提示。我们设计了一个行为分析器用于将回合结果归因于特定提示组件，以及一个变异器用于提出针对性的提示修订建议，然后通过环境部署进行验证。我们在BALROG基准测试中的所有五个BabyAI任务上进行了评估，在普通和有引导的提示初始化条件下，将我们的流程与BALROG的RobustCoTAgent进行了比较。优化方法在所有任务和条件下均持续提升了性能，且无需更新模型权重。在PutNext（一项多步协调任务，RobustCoTAgent的成功率为0%）上，我们的框架通过使用相同的底层LLM和优化后的提示，达到了高达72.5%的成功率。这些结果表明，多智能体框架结合自动提示优化能够增强LLM，而无需微调或大量人工监督。

## Abstract
LLM agents in interactive environments are highly sensitive to their prompts, yet prompt engineering remains a manual, task-specific process. We introduce an automated prompt optimization framework for LLM agents that decomposes the observation-to-action pipeline into a goal-conditioned descriptor agent and an action selection agent, and iteratively refines each module's prompt through an LLM-driven evolutionary loop guided by environment returns. We propose a behavior analyzer to attribute episode outcomes to specific prompt components, and a mutator to propose targeted revisions to the prompt, before validating them through environment rollouts. We evaluate on all five BabyAI tasks in the BALROG benchmark, comparing our pipeline against BALROG's RobustCoTAgent under both plain and guided prompt initializations. Optimization improves performance consistently across tasks and conditions, without requiring updates to the model weights. On PutNext, a multi-step coordination task where the RobustCoTAgent achieves 0% success, our framework reaches up to 72.5% success rate using the same underlying LLM with optimized prompts. These results suggest that a multi-agent framework, combined with automatic prompt optimization, enhances LLMs without the need for fine-tuning or extensive human supervision.

#### 2. 方法论：RAPOA框架

- **核心思想**：提出一个名为 **RAPOA**（Reward-driven Automatic Prompt Optimisation for Agentic systems） 的框架。该框架的核心是将传统的单一LLM智能体分解为两个角色：“**描述器**（Descriptor Agent）”和“**动作选择器**（Action Selection Agent）”，并通过一个**环境奖励驱动的进化循环**，自动迭代优化这两个模块的提示。

- **关键技术细节**：
    1.  **智能体架构分解**：
        - **描述器 (Ades)**：接收原始环境观察（文本化），并专注于提取与当前任务目标直接相关的信息，生成一个精简、目标导向的场景描述。
        - **动作选择器 (Asel)**：接收任务指令和描述器生成的摘要，并基于此做出单步动作决策。
        - 这种**“感知-行动”分离**（类似强化学习中的Actor-Critic）减少了单一智能体同时处理状态解析、策略适应和动作选择的负担。

    2.  **自动提示进化循环**：
        - **行为分析器 (Behavior Analyzer)**：分析智能体与环境交互生成的轨迹（包括成功和失败的回合）。其核心功能是**归因**，即把失败归因到特定的子模块（描述器或动作选择器）和具体的提示内容上，并输出一个按严重程度排序的问题和修改建议列表。
        - **变异器 (Mutator)**：接收行为分析器的诊断结果，对目标子模块的当前提示进行精确的修改（新增、修改或删除特定指令），生成新的候选提示。
        - **两阶段评估与接受标准**：候选提示必须通过两个评估阶段才能被接受：
            - **优化阶段**：在新的环境种子（Seeds）上评估，其平均表现必须比当前提示高出一个阈值 `δ`。
            - **选择阶段**：在固定的、未在优化中使用的选择种子上评估，确保改进的泛化性。
            - 引入了**高选择压力（HSP）** 和**低选择压力（LSP）** 两种评估策略，HSP更严格，LSP更宽松，用以控制进化的探索程度。

#### 3. 实验设计

- **环境与数据集**：使用 **BALROG** 基准测试中的所有五个 **BabyAI** 任务族：`GoTo`, `PickUp`, `Open`, `PutNext`, `PickUpSeqGoTo`。这些任务覆盖了从简单导航到多步协调的不同难度。环境提供自然语言任务描述和逐步的文本化观察与奖励。
- **基准方法**：主要对比方法是 BALROG 中的 **RobustCoTAgent**，它是一个单智能体、思维链（CoT）驱动的基线。
- **对比设置**：
    - **智能体类型**：本文提出的 **SPA**（Split Perception Action Agent，即分解后的描述器+动作选择器） vs. 基线 **BALROG**（单智能体）。
    - **提示初始化**：**普通 (Plain)** 和 **有引导 (Guided)** 两种。Guided提示包含了针对具体任务的专家知识（如获胜条件），Plain则只包含最少指令。这用于测试优化对不同初始质量的提示的有效性。
    - **优化策略**：不同阈值 `δ`（0.00, 0.02, 0.05, 0.10）、高/低选择压力（HSP/LSP）和“总是接受”（Always-accept）策略。
    - **消融实验**：限制优化器仅优化**描述器**或**动作选择器**。
    - **跨任务泛化实验**：将针对一个任务优化好的提示，直接应用于其他任务进行评估。

#### 6. 主要结论与发现

1.  **性能显著提升**：RAPOA优化框架在所有任务上**一致优于**其各自的非优化基线。对于最具挑战性的 `PutNext` 任务，优化后的SPA智能体从接近0%的成功率提升至高达 **72.5%**（LSP, Guided），而BALROG基线即使在优化后仍为~0%。
2.  **分解架构的鲁棒性**：SPA智能体（分解为描述器和动作选择器）本身对提示质量（普通 vs. 引导）的**敏感性低于**单智能体BALROG。即使在普通提示下，SPA的表现也足以媲美经过引导的单智能体。
3.  **接受标准至关重要**：使用“总是接受”策略会导致提示在优化后期**发生退化**（Catastrophic Forgetting），因为优化器无法区分好的和坏的变异。设置适当的接受阈值 `δ` 是保持性能稳定提升并防止退化的关键。
4.  **提示优化的泛化性**：针对组合型和困难任务（如`PutNext`）优化得到的提示，能更好地泛化到其他任务，而针对孤立任务（如`Open`）的优化则泛化性较差。

#### 7. 优点

- **方法创新性**：将**多智能体架构分解**与**强化学习式的信号（环境奖励）** 驱动的**自动提示进化**相结合，这是一个新颖且有效的思路，区别于仅在固定数据集上优化提示的工作。
- **实用性**：无需更新模型权重，适用于任何提供文本接口的LLM（包括闭源模型），降低了应用门槛。通过自动优化，极大地减少了对人工提示工程的依赖。
- **理论清晰**：使用行为分析器进行**归因**是该方法的一个亮点，使得优化过程具有可解释性，并能让优化器精确修改问题相关的模块。
- **实验严谨**：通过两阶段评估和固定种子，保证了评估的公平性和结论的可靠性。对接受标准、选择压力等超参数进行了细致的消融研究，揭示了其影响机制。

#### 8. 不足与局限

- **领域与模型有限**：实验仅在一个领域（BabyAI网格世界）和一种LLM（GPT-OSS20B）上进行，**泛化性**有待验证。未来需要在更多样化的环境（如更复杂游戏、机器人控制等）和更大规模的LLM上进行测试。
- **架构局限性**：仅实例化了一种优化架构（描述器+动作选择器）。框架不应仅限于此，可以扩展到包含记忆模块、规划模块等更复杂的多智能体设计。
- **优化成本**：优化过程本身计算成本较高，特别是对于困难任务。每次变异都需要环境交互来评估，这可能限制了其在实时或计算资源受限场景下的应用。
- **提示质量依赖**：优化的最终效果仍受限于初始提示的质量。虽然可以从“普通”提示开始改进，但一个高质量的“引导”提示能提供一个更好的起点，取得更优的最终结果（如`PutNext`任务所示）。
- **行为分析器的偏差风险**：行为分析器本身也是一个LLM，其诊断和归因可能存在偏差或不准确，这会直接影响后续变异的有效性。论文没有深入探讨或量化这种偏差风险。
