## 摘要
视觉语言模型（VLM）智能体正越来越多地部署在交互式游戏环境中。然而，针对VLM智能体的游戏基准测试通常为每个（智能体，游戏）对报告单一的首轮尝试分数，侧重于单智能体单人游戏，并且缺乏统一的协议来公平评估异构智能体类别（商业VLM、开放权重VLM和专用游戏策略）。我们通过OmniGameArena填补了这些空白，这是一个包含十二个新构建的虚幻引擎5游戏的实时基准测试，涵盖单人游戏（7个）、玩家对战（3个）和合作游戏（2个），并具有统一的动作接口；此外，我们还提出了改进动态曲线（IDC），这是一种智能体反思机制，其中使用工具的反省型LLM在多轮迭代中自主优化受限的技能提示。除了冷启动排行榜分数外，IDC还为每个（智能体，游戏）对揭示了另外两个可观测指标：分数在反思轮次中的演变方式，以及所学技能在保留任务变体上的表现。我们在冷启动排行榜上报告了十二个VLM智能体的这些指标，并在IDC下报告了四个顶级智能体的指标。

## Abstract
Vision-language model (VLM) agents are increasingly deployed in interactive game environments. Yet game benchmarks for VLM agents typically report a single first-attempt score per (agent, game) pair, focus on single-agent Solo play, and lack unified protocols for evaluating heterogeneous agent classes (commercial VLMs, open-weight VLMs, and specialized game policies) on the same footing. We address these gaps with OmniGameArena, a real-time benchmark of twelve newly built Unreal Engine 5 games spanning Solo (7), PvP (3), and Coop (2) with unified action interfaces, and the Improvement Dynamics Curve (IDC), an agentic-reflection harness in which a tool-using reflector LLM autonomously refines a bounded skill prompt across multiple rounds. Beyond cold-start leaderboard scores, IDC exposes two additional observables for each (agent, game) pair: how the score evolves across reflection rounds, and how the learned skill behaves on held-out task variants. We report these observables for twelve VLM agents on the cold-start leaderboard and four top agents under IDC.

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 构建新游戏避免预训练泄露，统一动作接口（键盘鼠标、手柄）使不同类别智能体可在同等环境下运行。
- 提出IDC，一种智能体反射机制，不使用权重更新，而是通过LLM自身的反思能力，在多轮迭代中自动优化**技能提示**（自然语言策略描述），并记录每轮分数序列（IDC曲线），同时测试技能在未见任务变体上的迁移能力。

### 关键技术细节
1. **OmniGameArena游戏套件**：
   - 12个UE5游戏，每个游戏进度归一化到[0,1]连续分数。
   - 游戏设计七维能力：视觉感知、空间导航、反应、记忆、规划、对抗性、合作性。

2. **IDC框架**（图3）：
   - **经验获取模块**：在每轮中，智能体在当前技能提示\(m_r\)下运行K个回合，得到轨迹集合\(\tau_r\)和平均分数\(S_r\)。第0轮为空技能\(m_0=\emptyset\)作为冷启动基线。
   - **反射模块**：反射器LLM（使用工具）在接收到新轨迹和持久状态后，自主执行四个阶段：
     - **探索**：通过沙盒只读工具检查轨迹（如`list_dir`, `read_text`, `read_image`, `grep`），由反射器自行决定检查什么。
     - **诊断**：提交明确的失败原因列表（`submit_diagnosis`），区分原因和处方。
     - **验证**：提出更新后的技能，由独立LLM评判（`validate_skill`），拒绝记忆地图内容或与诊断矛盾的提案，最多迭代5次。
     - **提炼**：最终确定技能\(m_{r+1}\)，并可选择编辑持久化的经验笔记本。
   - **持久化模块**：跨轮携带三个工件：
     - 经验笔记本（LLM读取的日志，≤2000 tokens）
     - 已验证的技能（当前技能提示\(m_r\) + 最佳技能缓存\(m_{r*}\)用于回滚）
     - 分数曲线（每轮分数序列）
   - **最佳技能回滚**：当新轮分数低于最佳分数的α倍（α=0.5）时，回滚到最佳技能，防止灾难性漂移。
   - **IDC曲线**：经过R轮得到\([S_0,S_1,...,S_R]\)，反映学习动态。

3. **每回合循环**：
   - 滑动窗口保存最近L个观察-响应对；当前帧附加到历史后。智能体基于系统指令、技能提示\(m\)、历史帧和当前帧输出动作。

## 6. 论文的主要结论与发现

1. **冷启动排行榜**：无单一模型主导，商业VLM远超开源VLM和专用策略。GPT-5.5在多数Solo游戏中领先，Claude Opus 4.6在CueChase大幅领先，Gemini 3.1 Pro在MonsterShoot领先。
2. **PvP**：存在非传递性，MidlineClash中Kimi K2.5击败Claude Opus 4.6，而Claude在Solo更强。
3. **Coop**：分数远低于Solo，最强模型仅0.368分；开源VLM得分为0，显示LLM-LLM协调是未解决的挑战。
4. **IDC发现**：
   - 所有模型通过反射提升，但峰值常出现在中间轮次而非最终轮（需要回滚机制）。
   - 起源任务提升与变体迁移能力**解耦**：Opus 4.7在LastStand上origin提升最大(+201%)但在所有变体上负迁移；GPT-5.5提升最小(+130%)却在三个变体上全部正向迁移，说明技能风格（保守vs.动态）决定了泛化性。
5. **延迟影响**：任务依赖。在静态生存任务中延迟无害甚至有益；在动态射击任务中延迟严重降低性能；在对称PvP中延迟压缩分数差距但不改变排名；在合作吞吐任务中延迟大幅减少动作预算。

## 7. 优点：方法或实验设计上的亮点

1. **游戏独创性**：12个游戏均为新构建，并通过污染验证，大幅降低预训练泄露风险。
2. **统一评估协议**：单一动作接口（键盘鼠标、手柄适配器），确保不同智能体在同等条件下比较。
3. **IDC新颖性**：不仅报告单次分数，还提供了**分数演化曲线**和**技能迁移能力**，揭示单次指标无法反映的学习动态和泛化特性。
4. **反射器自主性**：不预设反思脚本，反射器通过工具自主决定检查内容，更接近真实的自我改进。
5. **最佳技能回滚**：应对非单调反射的有效机制，防止技能恶化。
6. **延迟等价控制**：提出PDQ和LCRT两种时钟模式，隔离网络延迟和纯模型延迟的影响，使评估更公平。
7. **多模式覆盖**：同时包含Solo、PvP、Coop，全面测试不同能力维度。

## 8. 不足与局限

1. **IDC范围有限**：仅覆盖LastStand和SharedFloor两个游戏（各3个变体）和4个模型，结论的泛化性需更多游戏验证。
2. **单技能格式**：反射器维持单个技能提示并完全替换，而非类似Voyager的增量技能库，可能限制知识的累积。
3. **玩家与反射器共享模型**：使用相同模型作为玩家和反射器，未探究不对称设置（如小模型玩家+大模型反射器）是否能带来更好的改进。
4. **资源开销未讨论**：商业VLM API成本高，反射过程增加多轮推理费用，实际部署可行性未评估。
5. **变体设计可能不完整**：仅3个变体，可能未覆盖所有关键任务变化（如PvP变体未测试）。
6. **专用策略适配问题**：NitroGen和Open-P2P使用其原生接口，未针对OmniGameArena优化，可能不是最优表现（如NitroGen被设计用于游戏手柄，但部分UE5游戏可能更适合键鼠）。
7. **PvP反射未单独评估**：IDC避免了PvP，因为对手行为变化会干扰自主学习信号的归因，但实际对抗中的自适应也是重要能力，未被覆盖。
