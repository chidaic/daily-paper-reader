## 摘要
智能体强化学习使大语言模型智能体能够从环境奖励中持续改进，但其产生的策略无法系统性地积累跨任务泛化的可重用策略。模块化技能可以提供此类可重用策略，然而现有的技能增强强化学习方法将技能创建与策略优化解耦，可能导致采用与进化策略相冲突的技能。受Anthropic的Skill Creator启发，我们提出了ReSkill，一种在强化学习循环中进行技能创建的框架，协调技能演化与策略学习。ReSkill利用GRPO的组群结构自然地嵌入三种机制，仅需微小的额外开销：（1）基于断言的技能创建器，从过往经验中诊断失败并提出基于条件的、触发式的技能修订；（2）组内轨迹采样，支持技能版本的可控比较，捕捉哪个版本最能支持策略的持续学习；（3）带自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择的探索与利用。在多个领域，ReSkill持续优于现有基于记忆和技能的强化学习方法，在未见任务上提升最大。对技能生命周期的分析显示，随着策略改进，技能会被自动创建、测试、完善和修剪，展示了调和后的技能-策略协同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

- **研究背景**：大语言模型（LLM）智能体能够通过强化学习（RL）从环境奖励中持续改进，但其产生的策略无法系统性地积累跨任务泛化的可重用策略。模块化技能（modular skills）可以提供此类可重用策略，但现有技能增强RL方法（如Anthropic的Skill Creator、SkillRL等）通常将技能创建与策略优化解耦：技能在静态策略或离线场景下创建，再注入到训练中，导致技能与进化中的策略可能发生冲突。
- **核心问题**：如何将技能创建过程与策略学习过程**在线调和**，使技能能够随着策略的演化被动态测试、选择、精炼和剪枝，避免解耦带来的技能-策略冲突，同时不增加额外的采样开销。
- **整体意义**：提出了ReSkill框架，首次将Anthropic人机循环中的Skill Creator思想嵌入到在线RL训练循环中，利用GRPO的组结构实现零额外采样成本的技能-策略协同进化，为智能体持续学习提供了新范式。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 利用GRPO的组结构（每步对一个任务采样G条轨迹，共享同一策略）进行**组内技能版本分配**：在同一GRPO组中，不同轨迹可使用不同技能版本（旧版本 vs 新候选版本），通过组内对比实现可控评估。
- 每条轨迹同时承担三个角色：① 策略优化（GRPO梯度）；② 失败诊断与技能创建（填充经验池）；③ 技能版本评估（更新汤普森采样后验）。
- 使用**断言驱动的技能创建器**替代人类反馈，从持续演化的经验池中自动诊断失败模式、提出技能修订。
- 使用**自适应折扣的汤普森采样**在技能版本选择中平衡探索与利用，并考虑策略演化带来的旧证据过时问题。

### 2.2 关键技术细节

#### （1）组内技能版本分配与三重职责
- 对每个GRPO组，每个轨迹i独立从汤普森采样后验中采样版本vi∈{新,旧}，生成轨迹τi，计算奖励ri。
- 优化目标：J_{ReSkill} = E[ L_{GRPO}(θ; {τi}) ]，其中版本分配融入期望。
- 三个功能同时实现：策略更新、经验收集（用于技能创建）、版本评估（更新后验）。

#### （2）经验池与断言驱动的技能创建
- **经验池R**：持续收集轨迹（成功/失败对），标注版本标签和奖励，LIFO结构，按任务和版本保持多样性。
- **条件式技能加载**：每个技能含触发条件（general/beginning/action_pattern），仅在触发时加载到上下文。
- **断言集A**：由LLM动态维护的规则谓词集合，对全部轨迹计算通过率，用于量化失败模式。
- **技能创建流程**：对比分析（成功vs失败，新技能vs旧技能）→ 断言分级与诊断 → 语义聚类 → 技能推荐（增/改/删）→ 技能作者编写新版本 → 触发率验证（保证候选技能有足够激活）。

#### （3）自适应折扣汤普森采样
- 将新旧两个技能版本视为两支臂，各自维护Beta后验(α,β)。
- 在每步t，从后验中估计Pr(p_new > p_old)，作为分配概率π_t(new)，经截断确保最小探索。
- **自适应折扣更新**：每步后，对后验伪计数进行折扣：α_v ← w_t·α_v + m_t, β_v ← w_t·β_v + (n_t - m_t)，其中折扣权重 w_t = 1/(1 + n_t/M)，M为记忆参数。
  - 当该版本本步采样数n_t大时，w_t小（积极遗忘旧证据）；n_t小时，w_t接近1（保留历史）。
- **M的在线估计**：通过最大化已完成评估周期的序列预测似然来估计，无需手动调节。
- **接受/拒绝决策**：每个周期结束时，若E[p_new] > E[p_old]则接受新版本，否则回退。

### 2.3 算法流程（文字说明）
1. 初始化策略、空技能库、空经验池，Beta后验为(1,1)，记忆参数M初始为无穷。
2. 每训练步t：
   - 从当前后验计算分配概率，对G条轨迹独立采样版本vi。
   - 生成轨迹，计算奖励，用J_{ReSkill}更新策略。
   - 将轨迹加入经验池。
   - 更新各版本折扣后Beta后验。
   - 若t是演化周期的倍数：
     - 根据折扣后验接受或拒绝当前新版本。
     - 从经验池中诊断失败，创建新候选版本S_new。
     - 重置后验，开始下一轮测试。
3. 返回优化后的策略和技能库。

## 6. 主要结论与发现

1. **ReSkill在所有基准上一致超越现有方法**：在ALFWorld（4B/8B）上总体准确率89.8%/92.7%，比最强RL基线SkillRL高5.9%/6.9%；在Search上47.6%/49.8%，比SkillRL高2.5%/2.3%。
2. **在未见任务上增益最大**：ALFWorld未见任务8B下ReSkill达95.3% vs SkillRL 82.6%（差距12.7%）；Search未见分裂的增益约为已见分裂的2.1倍。表明调和进化保留了更泛化的技能。
3. **难度越高，增益越大**：ScienceWorld OOD任务ReSkill达48.8% vs SkillRL 26.9%；InterCode-SQL超难查询上63.6% vs 55.3%；WANDS 3-opt设置上86.6% vs 82.1%。
4. **调和机制至关重要**：自动接受所有技能（无测试）会导致训练中途崩溃，性能甚至低于无技能的GRPO；均匀分配则落后带汤普森采样的ReSkill。
5. **技能创建器选择影响有限**：使用基模型自身创建技能（弱于Claude 4.5 Sonnet）仅导致ALFWorld下降4.8%、Search下降0.5%，仍优于所有现有RL方法，说明收益主要来自调和机制而非创建器能力。
6. **技能生命周期自动演进**：早期建立基础技能（如搜索-验证、流程顺序），后期修剪已内化技能，转向精细化策略。
7. **跨域时效性适应**：冻结策略后仅靠技能创建就能快速适应新领域，说明调和训练培养了“遵循并受益于新技能”的通用能力。

## 7. 优点

- **方法创新性强**：首次在在线RL中实现技能创建与策略优化的协同，将GRPO组结构创造性用于技能版本对比，实现零额外采样成本的三重作用。
- **可解释性好**：技能生命周期可视化、案例研究展示技能如何指导推理（包括导致错误的元bug），有助于理解机制。
- **实验设计扎实**：覆盖5个领域、2种模型尺度、多个未见/困难分裂，并进行了充分的消融和敏感性分析；对比基线包含当前主流方法（MemRL、EvolveR、INSPO、SkillRL等）。
- **效率高**：条件触发机制只加载相关技能，保持上下文紧凑；额外训练开销仅5-15%；无需额外采样预算。
- **实用性强**：自适应记忆参数M无需手动调节，演化频率和技能库大小默认值在多数场景表现稳定。
- **开源可复现**：提供了代码网站、完整提示模板、超参数表，所有基线使用原始公共实现。

## 8. 不足与局限

1. **演化频率仍是超参数**：需要根据训练总步数比例设置，低进化频率在短训练周期下导致性能下降（如ALFWorld上Ke=20表现差），需要自适应进化调度。
2. **技能调用机制灵活性有限**：当前触发式加载（一旦匹配即注入上下文）给模型较少控制权；模型有时将技能名误作环境动作（元bug），或在不适用时过度应用。建议探索将技能作为显式工具调用。
3. **小模型认知负荷**：较小的模型（4B）有时混淆技能指导与环境动作，或在不适用的情境下仍使用技能，表明技能增强RL需要训练策略将技能视为推理指导而非动作指令，例如通过格式感知奖励塑形。
4. **技能库容量负担**：增加技能到上下文会增加基模型认知负荷，本研究固定库大小为8，更大库大小（16）在ALFWorld上性能下降（图9）。
