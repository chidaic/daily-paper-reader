## 摘要
随着大型语言模型（LLM）的广泛部署，它们越来越需要与其他AI智能体协同工作，而非孤立运行。在这些场景中，有效协调要求智能体在不确定性下进行通信、共享信息和做出决策。我们引入了SMAC-Talk，这是星际争霸多智能体挑战（StarCraft Multi-Agent Challenge）的一个自然语言扩展，用于评估基于LLM的智能体在协作多智能体环境中的表现。该环境具有几个关键特征，如分散控制、部分可观测性和长视野决策。SMAC-Talk包含一个自然语言通信通道，用于探测智能体的协调与信任。我们利用这个通信通道构建了不同的评估场景，包括一个包含欺骗性通信者的设置，该通信者试图仅通过沟通来干扰和欺骗盟友。我们提供了三个基准测试智能体，使用了Qwen3.5系列的四个模型，并研究了推理结构、记忆和模型规模如何影响智能体间的协调。我们将SMAC-Talk作为一个开放基准发布，以支持研究社区在协作多智能体场景中开发和评估LLM智能体。

## Abstract
As LLMs become more widely deployed, they are increasingly expected to work alongside other AI agents rather than operating in isolation. Effective coordination in these settings requires agents to communicate, share information and make decisions under uncertainty. We introduce SMAC-Talk, a natural language extension of the StarCraft Multi-Agent Challenge for evaluating LLM-based agents in cooperative multi-agent environments. The environment has several key features such as decentralized control, partial observability and long-horizon decision making. SMAC-Talk includes a natural language communication channel which is used to probe agent coordination and trust. We use this communication channel to construct different evaluation scenarios, including settings with an embedded deceptive communicator that tries to disrupt and deceive allies through communication alone. We provide three agents for benchmarking using 4 models from the Qwen3.5 family and study how reasoning structure, memory and model scale affect coordination between agents. We release SMAC-Talk as an open benchmark to support the research community in developing and evaluating LLM agents in cooperative multi-agent settings.

### 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：当前大多数LLM多智能体基准聚焦于纯文本生成或简化协调任务，缺乏类似多智能体强化学习（MARL）中SMAC那样的交互式、具身化（embodied）环境来评估LLM在部分可观测、分散控制、长时域决策下的协调能力。同时，现有基准未系统研究自然语言通信对协调的影响，以及智能体对欺骗性通信的鲁棒性和信任问题。
- **整体含义**：SMAC-Talk旨在填补这一空白，通过将经典MARL基准SMACv2扩展为自然语言环境，使基于LLM的智能体能够用文本观察、动作和通信进行协作，并设计欺骗性通信者场景来探测智能体的信任与抗干扰能力，为LLM在多智能体系统中的安全部署提供评估平台。

### 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：将SMACv2的数值观察与离散动作通过适配器转换为自然语言输入/输出，并添加通信层，保持环境的部分可观测性和分散控制特征。
- **关键技术细节**：
  - **观察-to-文本适配器**：将每个智能体的局部观察（自身状态、可见盟友、可见敌人）转化为结构化自然语言描述，包括单位类型、生命值、位置、距离等。启用通信时，盟友消息也会附加在观察中。
  - **文本-to-动作适配器**：将LLM输出的自然语言动作（如“move north”、“attack enemy 2”）映射为离散游戏动作。无效或无法解析的动作被替换为stop并记录。
  - **自然语言通信层**：智能体每步可广播一条消息，消息在下一时间步传递给可见的盟友，保持部分可观测性。环境不过滤或修改消息。
  - **智能体架构**：提出三种基准智能体：
    - **Zero-Shot Agent**：最小指令，无推理过程，使用非思考模式。
    - **ReAct Agent**：结构化思维链，包含游戏启发式（集火、定位、通信规则），保留2步历史，使用非思考模式。
    - **Reasoning Agent**：使用Qwen3.5内部思维模式（1024 token预算），自由推理，保留2步观察和动作历史。
  - **欺骗性通信者**：被指示不参加战斗，仅移动和通信，制造虚假敌情、建议劣质行动、假装意图等，在已知（KDC）和未知（UDC）两种设定下测试。

### 论文的主要结论与发现
1. **内部推理优于结构化推理**：Reasoning Agent（内部思维链）在所有模型规模和场景下一致优于Zero-Shot和ReAct。
2. **外部推理在通信下崩塌**：ReAct Agent在通信场景下表现急剧下降，原因可能是启发式与通信结构不兼容，提示设计对多智能体设置影响巨大。
3. **模型规模至关重要**：4B不足以可靠协调，9B是有效下限，27B和122B-A10B之间收益递减。大模型更能利用通信和抵抗欺骗。
4. **通信效果因智能体而异**：Zero-Shot和Reasoning从通信中获益有限，ReAct则受损。
5. **欺骗场景下大模型的鲁棒性**：在已知欺骗者（KDC）场景中，大模型显著优于未知（UDC），尤其在5v5中（122B-A10B推理：KDC胜率41% vs UDC 10%），表明知情告知能帮助大模型更好地依赖本地观察来折扣不可靠消息。

### 优点：方法或实验设计上的亮点
- **填补基准空白**：首个将经典MARL环境自然语言化并支持LLM的交互式多智能体协调基准，尤其包含自然语言通信和欺骗性对抗通信。
- **模块化设计**：观察、动作、通信适配器独立，推理无关，支持多种LLM后端（vLLM、Llama.cpp、Cerebras等），便于扩展。
- **系统化评估**：同时考量推理结构、记忆、模型规模和通信类型，揭示了提示架构在多智能体环境中的深远影响。
- **开源**：代码和全部提示变体公开，促进可复现研究。

### 不足与局限
1. **计算成本高**：运行LLM多智能体系统耗算力显著，小模型（4B）性能不足，限制了可复现性。
2. **动作错误指标不完美**：无效动作被stop替代并记录，可能掩盖模型真实能力，惩罚格式不统一但推理正确的模型。
3. **未探索推理预算影响**：固定1024 token思维预算，未研究不同深度对协调的影响。
4. **仅测试单一模型家族**：结果可能不泛化到其他LLM（如Llama、GPT-4等）。
5. **环境限制**：仅使用人族单位和Very Easy敌方难度，未探索更多种族、难度和单位组合。
6. **ReAct崩溃根因未明**：通信导致ReAct性能骤降的机制未深入分析，可能源于推理结构或通信引导方式。
7. **欺骗场景依赖性**：欺骗性通信者受限于域内语言，未能测试更隐蔽或利用提示注入等攻击方式。
