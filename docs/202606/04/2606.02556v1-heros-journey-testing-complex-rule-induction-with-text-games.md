## 摘要
我们提出英雄之旅（HERO'S JOURNEY），这是一个用于在目标导向的情节任务中进行规则归纳的基准测试，其中代理必须从示范中推断隐藏规则，并通过多步执行来应用它们。英雄之旅涵盖了属性归纳和程序归纳两大类的八个任务，每个任务有四种结构规则形式、可控的词汇基础以及可辨识性条件。评估最先进的大型语言模型时，我们发现模型表现出规则归纳的迹象，但这种能力有限且在不同任务间不均衡。同时，过程执行给模型增加了执行瓶颈，而表面语义影响甚微。针对归纳的导向方法在属性任务上提升了性能，但在程序任务上没有可靠改进，表明程序归纳的差距仍然是一个开放的挑战。

## Abstract
We introduce HERO'S JOURNEY, a benchmark for rule induction in goal-directed episodic tasks, where agents must infer hidden rules from demonstrations and act on them through multi-step execution. HERO'S JOURNEY covers eight tasks across attribute and procedural induction families, each with four structural rule forms, controllable lexical grounding, and identifiability conditions. Evaluating state-of-the-art LLMs, we find that models show evidence of rule induction, but the ability is limited and uneven across tasks. Meanwhile, process execution adds an execution bottleneck for models, whereas surface semantics has minimal effect. Induction-specific steering methods improve performance on attribute tasks but show no reliable gains on procedural tasks, suggesting the gap in procedural induction remains an open challenge.

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：设计一个可配置、可控的文本游戏环境，其中任务所需规则隐藏在示范中，代理需同时完成归纳（从示范中学习规则）和程序执行（按规则正确执行动作序列）。通过控制规则结构、词汇语义和可识别性条件，实现精细评估。

- **关键技术细节**：
  - **任务构建**：
    - **过程（Process）**：一个目标网络`(G, ≺, ℓ)`，定义子目标集合、偏序关系和动作-参数赋值。固定基序：`go(item_loc) → buy(item) → go(entity_loc) → defeat(entity)`。
    - **规则规范（Rule Specification）**：独立属性集`I = V₁ × V₂`，隐藏映射：
      - 属性归纳（Attribute Induction）：`f: I → P`，输出物品属性（如大小、颜色）。
      - 程序归纳（Procedural Induction）：`σ: I → A × O`，输出动作类型和次序。
    - **任务规范（Task Specification）**：绑定规则函数、过程模板、源/生成集划分，并满足可识别性条件。
  - **四种规则结构形式**（适用于属性和程序两种任务族）：
    1. **加性（Additive）**：`f(v₁,v₂) = ϕ₁(v₁) + ϕ₂(v₂)`，每个属性独立贡献一个数值。
    2. **组合（Compositional）**：`f(v₁,v₂) = (s(v₁), c(v₂))`，每个属性控制不同输出维度。
    3. **条件（Conditional）**：`f(v₁,v₂)`由v₁所在“体制”决定v₂控制哪个维度。
    4. **覆盖（Override）**：当v₂等于特例`u*`时，输出固定值；否则由v₁决定。
  - **可识别性条件**：确保源示范足以唯一恢复目标规则（例如组合任务要求示范形成二部连通图）。
  - **评估指标**：
    - **ECSR（效率校准成功率）**：`success_rate × norm_eff`，其中`norm_eff`衡量尝试次数相对暴力枚举的效率（1次尝试为1，暴力枚举为0）。
    - **RV（规则言语化得分）**：模型用自然语言描述规则，由LLM裁判（Qwen3.5-27B）在0-2分尺度上评分，经人类标注验证（Krippendorff's α=0.795）。

## 6. 论文的主要结论与发现

1. **模型规则归纳能力有限且不均衡**：最强模型（GPT系列）在部分属性任务上接近人类ECSR（如A-Comp达0.93 vs 人类0.75），但整体落后：人类平均ECSR比模型高13%，RV高30%。
2. **ECSR与RV正相关但程序任务更弱**：全模型×任务Spearman ρ=0.54 (p<0.001)，但程序任务ρ=0.36 (p=0.08)，说明言语化与行为一致性在程序领域更差。
3. **过程执行是额外瓶颈**：多数模型在QA（直接回答）上表现优于情节执行，格式差距显著（GPT-5.4-mini属性任务Δμ=0.096, 程序任务Δμ=0.268），表明正确识别规则并不保证成功执行。
4. **表面语义影响极小**：词汇条件（语义 vs. nonce）对GPT-5.4-mini无显著影响，Qwen仅在属性任务中nonce略优，说明模型未依赖预训练知识。
5. **模型有时依赖抄袭而非归纳**：部分任务（如A-Cond）ECSR随覆盖率先升后降，实际跟踪的是“从示范中拷贝”的上下文偏差曲线，而非规则可识别性。
6. **现有方法仅对属性任务有效**：IDEA和HR能提升属性任务ECSR，但对程序任务无可靠改进，程序归纳仍是开放挑战。

## 7. 优点

- **设计系统且可配置**：规则结构、词汇、可识别性条件均可独立控制，支持精确归因。
- **首次引入多步执行瓶颈评估**：不仅测试规则识别，还测试规则驱动的行为序列，更贴近实际代理场景。
- **原则性可识别性条件**：确保示范足以确定目标规则，使成功可归因于规则归纳而非其他一致假设。
- **双指标互补**：ECSR（行为效率）和RV（言语理解）交叉验证，揭示模型在“能做但说不清”或“能说清但做不对”上的脱节。
- **消融与干预实验全面**：词汇条件、覆盖率、格式差距、方法对比等多维度分析，结论稳健。
