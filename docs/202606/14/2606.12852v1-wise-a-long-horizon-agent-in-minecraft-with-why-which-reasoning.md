## 摘要
通过采用大语言模型增强的分层方法，在Minecraft等环境中开发通用具身智能体取得了快速进展。尽管前景看好，但低级控制器常常因重复执行失败而成为性能瓶颈。我们认为，一个关键限制不仅是缺乏情景记忆，还在于将“什么-哪里-何时”记忆与“哪个-为什么”推理相分离。为了解决这个问题，我们提出了WISE（Why-Which Informed Semantic Explorer），一种长周期智能体框架，其增强的低级控制器配备了一个因果事件图，通过明确的因果结构将观察结果与任务相关性联系起来，增强了情景记忆。与之前依赖特征相似性进行检索的MrSteve等工作不同，WISE在视角变化下能够实现鲁棒的回忆，并支持通过因果推理进行机会性的任务重排。在此基础上，我们提出了一个机会性任务调度器，当检测到因果相关的机会时，动态地重新确定子任务的优先级。我们还为WISE配备了一种多尺度渐进探索策略，为下游推理提供空间上全面的观察。实验表明，WISE在长周期稀疏任务中大大提高了任务成功率和效率，特别是在需要自适应决策的场景中。

## Abstract
Rapid advances have been made in developing general-purpose embodied agent in environments like Minecraft through the adoption of LLM-augmented hierarchical approaches. Despite their promise, low-level controllers often become performance bottlenecks due to repeated execution failures. We argue that a key limitation is not only the lack of episodic memory, but also the decoupling of \textit{what-where-when} memory from \textit{which-why} reasoning. To address this, we propose \textbf{WISE} (Which-Why Informed Semantic Explorer), a long-horizon agent framework with an enhanced low-level controller equipped with a Causal Event Graph that augments episodic memory with explicit causal structure linking observations to task relevance. Unlike prior work such as MrSteve, which relies on feature similarity for retrieval, WISE enables robust recall under viewpoint changes and supports opportunistic task reordering through causal reasoning. Building on this memory, we propose an Opportunistic Task Scheduler that dynamically re-prioritizes subtasks when causally relevant opportunities are detected. We further equip WISE with a multi-scale progressive exploration strategy to provide spatially comprehensive observations for downstream reasoning. Experiments show that WISE largely improves task success and efficiency on long-horizon sparse tasks, particularly in settings requiring adaptive decision-making.

## 1. 核心问题与整体含义

- **研究动机**：在 Minecraft 等开放世界中构建具身智能体以解决长周期稀疏奖励任务仍面临根本性挑战。现有 LLM 增强的分层框架将任务分解为高层规划（生成子目标）和低层控制（执行子目标），但低层控制器因重复执行失败成为主要瓶颈。  
- **核心问题**：现有方法不仅缺乏情景记忆（what-where-when），更关键的是将记忆与**因果推理**（why-which）相分离。记忆存储不包含观察与任务的因果联系，导致检索在视角变化时脆弱，且智能体无法利用偶然遇到的机会来重排子任务。  
- **整体含义**：为了在稀疏、开放环境中实现高效自适应决策，需要一种闭式回路架构，将探索、记忆和规划协同起来，让低层控制器不仅能记住“什么、哪里、何时”，还能理解“为什么这个观察重要”以及“它使哪个未来任务成为可能”。

## 2. 方法论

### 2.1 核心思想
提出 **WISE（Which-Why Informed Semantic Explorer）**，一个统一框架，包含三个核心模块：
- **因果事件图（Causal Event Graph）**：给情景记忆注入显式的因果结构（通过 VLM 提取实体、因果边、共现关系）。
- **机会性任务调度器（Opportunistic Task Scheduler）**：基于因果相关性动态重排子任务优先级。
- **多尺度渐进探索**（Multi-Scale Progressive Exploration）：分三层次高效覆盖环境，为下游推理提供丰富观测。

### 2.2 关键技术细节

#### (a) 记忆模块
- **短期几何记忆**：类似 MrSteve 的 Place Event Memory（PEM），使用 MineCLIP 编码视频帧，DP-Means 聚类形成事件簇，余弦相似度检索。
- **因果事件图**：通过 VLM（GPT-4o）异步处理关键帧（混合聚类+熵提取），提取实体（牛、树等）、因果边（cow → CAN_OBTAIN → beef）和共现边（CO_OCCURS_WITH）。实体节点关联空间坐标。
- **两级检索**：
  - 短期层：视觉相似度检索。
  - 语义层：图查询，沿因果边找到相关实体并返回位置。
  - 加权融合公式：`Score(x, τ) = λ·sim(e_x, Enc_t(τ)) + (1-λ)·CausalMatch(x, τ)`，其中 `CausalMatch` 为 1 当存在因果关系。

#### (b) 机会性任务调度器
- 当新任务到来或观察变化时，计算每个待执行子任务的优先级：
  ```
  Priority(τ_i, t) = w_u·Urgency + w_r·CausalRel + w_n·(1 - NavCost)
  ```
  - `Urgency`：任务时序紧迫性。
  - `CausalRel`：当前观察/记忆是否与 τ_i 有因果路径（如检测到牛则“获得牛肉”任务因果相关度高）。
  - `NavCost`：归一化导航成本。
  - 权重设为 `w_u=0.3, w_r=0.5, w_n=0.2`，强调因果主导。
- 调度器按优先级重排任务队列，若最高优先级变化则抢占执行。

#### (c) 多尺度渐进探索
- **全局层**：四叉树分割环境，节点评分 = α·Depth + β·Dist，导向粗粒度未探索区域。
- **区域层**：前沿点检测，评分考虑覆盖增益、信息新颖性、导航成本。
- **局部完成层**：Voronoi 分解识别内部未探索间隙，评分考虑面积和距离。
- 三者联合实现高效、边界感知且局部完备的探索。

### 2.3 算法流程（文字说明）
1. 收到新任务 τ。
2. 从短期记忆和图检索候选。
3. 若存在候选（视觉或因果匹配），评分选择最佳，导航并执行。
4. 若调度器检测到因果相关的未完成子任务（如发现牛），重排任务并立即执行。
5. 否则，切换到多尺度探索模式，收集新观测并更新记忆。

## 6. 主要结论与发现

- **探索效率**：WISE 在真实 128×128 地图上 10,000 步内达到 97% 覆盖率，远超 MrSteve（67%）和 Steve-1（19%）。多尺度层级中每一层都贡献了互补增益。

- **非顺序稀疏任务（ABC）**：WISE 成功率 77% vs. MrSteve 33%，平均步数 4,620 vs. 8,040。机会性调度是主要驱动力，消除不必要的返程。

- **关键发现**：记忆与因果推理的分离是低层控制器瓶颈的根本原因；将“why-which”推理融入执行回路可显著提升长周期任务适应性和效率。

## 8. 不足与局限

- **VLM 依赖与成本**：每 episode 约 $0.48 的 GPT-4o 调用成本，对于大规模部署或离线环境可能不现实。异步运行不阻塞控制，但总 API 费用（整个实验约 $512）仍较高。
- **场景限制**：仅在 Minecraft 环境中验证，虽然其通用性可迁移，但未在更广泛的具身环境（如 Habitat、RLBench）中测试。
- **任务种类有限**：实验仅涉及资源收集型稀疏任务，未测试复杂交互（如合成、工具使用链条）或对抗性环境。
- **高层面计划假设**：WISE 假设高层 LLM 已分解好子目标序列，未涉及高层规划的协同优化；若高层分解错误，低层难以纠正。
- **动态实体处理**：对于移动实体（如动物），调度器可能因未能及时检测或实体离开而失败；论文提及部分失败案例源于此。
- **可复现性细节**：部分权重（如 w_u、w_r、w_n）通过经验设置，未提供自动化调优过程对一般场景的通用性；四叉树深度等参数针对特定地图尺寸设置。
- **长时间稳定性**：虽在 12,000-40,000 步内测试，但极端长时任务（如数小时）中记忆和探索的退化未分析。
