## 摘要
机器人足球是多智能体强化学习的一个具有挑战性的测试平台，因为它结合了部分可观测性、合作与对抗交互、稀疏奖励以及长时域战术行为。RoboCup 2D足球仿真（RCSS2D）提供了一个成熟的机器人足球平台，但其面向竞争的服务器-客户端架构难以直接与现代基于Python的MARL工作流一起使用。我们推出了R2D-RL，这是一个强化学习环境，通过共享内存通信和周期级同步将RCSS2D和基于HELIOS的玩家客户端连接到Python MARL接口。R2D-RL支持全场和基于场景的训练，具有可配置的对手、基本离散和混合参数化动作空间、动作掩码、基于预期控球价值（EPV）的奖励塑造以及并行执行。我们提供了前场进球场景和一个11对11的全场基准，以及基线结果。

## Abstract
Robot soccer is a challenging testbed for multi-agent reinforcement learning because it combines partial observability, cooperative and adversarial interaction, sparse rewards, and long-horizon tactical behavior. RoboCup 2D Soccer Simulation (RCSS2D) provides a mature robot-soccer platform, but its competition-oriented server-client architecture is difficult to use directly with modern Python-based MARL workflows. We introduce R2D-RL, a reinforcement learning environment that connects RCSS2D and HELIOS-based player clients to a Python MARL interface through shared-memory communication and cycle-level synchronization. R2D-RL supports full-field and scenario-based training with configurable opponents, Base discrete and Hybrid parameterized action spaces, action masks, expected possession value (EPV)-based reward shaping, and parallel execution. We provide front-goal scenarios and an 11-vs-11 full-field benchmark, together with baseline results.

- **整体意义**：R2D-RL 填补了 RCSS2D 生态系统与 Python MARL 工作流之间的空白，为研究全维度足球战术决策（部分可观察、稀疏奖励、长时域合作与对抗）提供了一个可重复、可扩展的标准化环境。

## 2. 方法论

- **核心思想**：通过**共享内存通信**和**周期级同步**，将 RCSS2D 服务器、修改后的 HELIOS 玩家客户端、教练和训练员进程与 Python 环境连接，使原生的比赛循环转换为步进式强化学习交互。
- **关键技术细节**：
  - **步同步通信协议**：基于序列计数器（obs_seq, act_seq, done_seq）实现“观察发布→动作分配→动作消费”的显式分离。只有 `play_on` 模式下的决策帧才触发环境步，其他模式（开球、任意球等）仍由原生规则和 HELIOS 处理。
  - **状态与观察**：
    - 集中状态（coach）：136 维向量，包含球的位置/速度、22 个球员的 xy 位置/速度/体向/队伍标识。
    - 局部观察（player）：97 维向量，包含自身信息（位置/速度/体力/控球标志）、球信息、最多 11 个对手和 10 个队友的 xy 位置/速度、游戏模式/边/守门员标志。
  - **动作空间**：
    - **Base 离散动作空间**：19 个高层语义动作（如 Shoot, Pass, Dribble, Move 等），由 HELIOS 行为模块内部生成底层命令。
    - **Hybrid 参数化动作空间**：直接暴露底层身体命令（turn, dash, kick, catch）及其连续参数，外加 HELIOS Fallback 和 empty 内部控制项。
  - **动作掩码**：基于 HELIOS 行为可执行性检查，为每个动作生成允许/禁止标志。Base 空间下，射门动作被赋予优先级，一旦可执行则屏蔽其他所有动作。Hybrid 空间根据踢球、接球等条件屏蔽对应命令。
  - **奖励设计**：
    - 团队级得分奖励：进球 +1，失球 -1。
    - 可选 MaxEPV 塑形奖励：基于预计算的 32×50 预期控球价值网格，只更新己方球员可踢球时的最大 EPV 值，正增益作为塑形奖励，进球时额外添加完成奖励。
  - **场景重置**：通过训练员客户端直接移动球和球员状态（位置、方向、速度），避免重启整个服务器，支持内建基准状态和用户自定义状态。
  - **并行执行**：多个环境实例通过 SLURM 分配 CPU 核心并行采样。

## 3. 实验设计

- **数据集/场景**：
  - **前场（front-goal）场景**：5 个递进难度的控球进攻任务（从空门射门到紧凑防守下的多攻击者协调），每种场景固定攻击方向朝右，最大步数分别为 100 或 300。
  - **全场 11v11 基准**：标准 RCSS2D 全场设定，左队由学习策略控制，右队使用 HELIOS Fallback 作为内建 AI，最大步数 3000，不进界死亡，目标事件触发发球但不终止。
- **基准设置**：
  - 固定对手：所有未被控制的球员（包括对手和非控制的队友）每步执行 HELIOS Fallback 动作。
  - 组件设置：每类场景测试四种配置（MaxEPV 塑形+动作掩码、仅掩码、仅 MaxEPV、无 EPV/掩码）。
- **对比方法**：
  - **Base 离散动作空间**：MAPPO（多智能体近端策略优化，使用 RMAPPO，共享 actor + 集中 critic）和 QMIX（单调值分解，使用循环智能体网络和集中混合器）。
  - **Hybrid 参数化动作空间**：ParaDQN（独立参数化 DQN，共享 Q 网络和参数 actor，无集中 critic 或值分解）。

## 6. 主要结论与发现

- R2D-RL 能够支持从简单射门到全场对抗的渐进式 MARL 实验，并行采样吞吐量在 14 个实例内线性增长（Base 空间 34.85M 步/天，Hybrid 空间 41.65M 步/天）。
- 在简单单攻击者任务（空门）中，所有算法均能达到近乎 1.0 的进球率，证明环境接口和基线训练流程的可用性。
- **动作掩码至关重要**：在压力射门（Blocked Shot）和多数多攻击者场景中，无掩码的 QMIX 和 ParaDQN 完全失败，有掩码时显著提升；MAPPO 在单攻击者场景中更具鲁棒性。
- **MaxEPV 塑形奖励**在复杂多攻击者场景（Compact Defense）中起到关键作用，几乎所有无 EPV 塑形的设置进球率接近零。
- **MAPPO 稳定性最好**，在前场和全场场景中均取得最高或接近最高的进球率/目标差；QMIX 种子方差大；ParaDQN（Hybrid 空间）在多攻击者场景中近乎失败，说明参数化动作空间在复杂任务中训练难度高。
- 全场基准下，30M 步后 MAPPO 目标差约为 -10.61，QMIX -19.00，ParaDQN -25.90，表明当前训练预算未达到高水平的团队配合，但已能观察到基础的球推进和多智能体协调迹象。

## 7. 优点

- **桥梁作用**：在不修改 RCSS2D 原始比赛逻辑的前提下，通过共享内存同步将其转化为强化学习可用的步进式环境，降低了 RoboCup 2D 在 MARL 领域的使用门槛。
- **兼容性与模块化**：保持与现有 HELIOS 行为生态的兼容，动作掩码、奖励塑形等组件可插拔，便于研究者复用和扩展。
- **丰富的前场渐进场景**：5 个递进攻防任务提供了可控的测试阶梯，有助于分析算法在增加协调要求时的表现退化原因。
- **动作空间灵活性**：同时提供高层语义动作（Base）和底层参数化动作（Hybrid），支持不同的算法研究需求。
- **并行采样效率**：实验显示在合理 CPU 核心分配下可达到数千万步/天的采样速率，满足中等规模训练需求。

## 8. 不足与局限

- **学习限于 `play_on` 模式**：死球场景（开球、任意球、球门球等）仍由原生规则和 HELIOS 处理，策略无法学习定位球战术，限制了战术完整性。
- **对手固定且单一**：所有实验仅使用 HELIOS Fallback 作为对手，未包含自我对弈或多对手池评估，不利于泛化性和鲁棒性研究。
- **动作空间依赖手写规则**：Base 空间的高层动作定义和掩码均基于 HELIOS 的专家知识，引入了归纳偏差。Hybrid 空间同样依赖手工设计的可执行条件。
- **训练预算有限**：全场 11v11 仅训练 30M 环境步，远未收敛到高水平，导致基线策略仍显著弱于内置 AI（目标差为负）。更长时间、更丰富课程的设计是未来必需。
- **ParaDQN 表现弱**：Hybrid 参数化动作空间在复杂任务中训练困难，可能归因于独立学习、无集中 critic 以及参数与离散命令联合探索的难度，但本文未深入分析改进方案。
