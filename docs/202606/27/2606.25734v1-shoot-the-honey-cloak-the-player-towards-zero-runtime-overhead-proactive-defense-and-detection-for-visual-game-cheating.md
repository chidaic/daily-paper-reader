## 摘要
视觉自瞄在第一人称射击（FPS）游戏中已成为一种严重的作弊威胁，因为它仅基于渲染帧而非游戏内存运行，从而规避了现有的反作弊防御。然而，现有防御未能提供端到端的解决方案：事后行为检测器无法实时保护比赛完整性，且越来越容易受到模仿人类的自瞄攻击；而主动运行时防御通常缺乏可问责性，产生大量开销，或需要侵入式系统集成。我们提出了AimTrap，这是首个针对视觉自瞄的端到端防御系统，它通过两种对抗性纹理机制将实时保护与赛后检测相结合。对抗性伪装纹理（ACT）将真实玩家隐藏于自瞄之外，而对抗性蜜罐纹理（AHT）则诱使自瞄锁定假目标，从而提供作弊的有力证据。为使其实用，AimTrap将可微分渲染与渲染期望结合，实现鲁棒的3D纹理合成、安全的纹理管理，以及一种新颖的蜜罐交互轨迹分析管道，用于准确归因作弊。在针对最先进视觉自瞄的真实游戏评估中，ACT实现了85.1%的防御成功率，AHT达到96.9%。与先前基线相比，AimTrap达到了极低的误报率，同时运行时开销可忽略不计。这些结果表明，AimTrap为对抗视觉自瞄提供了一种实用且有效的端到端防御方案。

## Abstract
Visual aimbots have emerged as a serious cheating threat in first-person shooter (FPS) games, as they evade existing anti-cheat defenses by operating only on rendered frames rather than game memory. However, existing defenses fail to provide an end-to-end solution: post-hoc behavior detectors cannot protect match integrity in real time and are increasingly fragile against human-mimicking aimbots, while proactive runtime defenses often lack accountability, incur substantial overhead, or require intrusive system integration.   We present AimTrap, the first end-to-end defense against visual aimbots that combines real-time protection with post-game detection using two adversarial texture mechanisms. Adversarial Camouflage Textures (ACT) hide real players from aimbots, while Adversarial Honeypot Textures (AHT) lure aimbots into locking onto fake targets, yielding strong evidence of cheating. To make this practical, AimTrap integrates differentiable rendering with Expectation over Renderings for robust 3D texture synthesis, secure texture management, and a novel honeypot-interaction trajectory analysis pipeline for accurate cheating attribution.   In real-game evaluation against a state-of-the-art visual aimbot, ACT achieves 85.1% defense success, AHT achieves 96.9%. Compared with prior baselines, AimTrap attains extremely low false-positive rates, while incurring negligible runtime overhead. These results show that AimTrap provides a practical and effective end-to-end defense against visual aimbots.

- **现有防御的局限性**：
  - 事后行为检测（post-hoc detection）：只能在比赛结束后分析行为，无法实时保护比赛完整性，且面对模仿人类的自瞄（如GAN生成轨迹、神经肌肉瞄准辅助）越来越脆弱。
  - 主动运行时防御（proactive runtime defense）：通过扰动显示帧或纹理来干扰自瞄，但通常引入大量性能开销（如每帧8.5秒或仅21–77 FPS），且缺乏可问责性（无法提供作弊证据），需要侵入式系统集成（如钩子、着色器修改）。
  - 现有纹理扰动方案仅支持2D平面，无法扩展到3D玩家模型和非平面建筑模型。
- **本文目标**：提出首个端到端（end-to-end）的视觉自瞄防御方案，同时实现：
  - **实时主动防御**（运行时零开销）
  - **赛后可解释检测**（提供证据）
  - **轻量级部署**（无需修改游戏引擎、无需每帧计算）

## 2. 方法：核心技术细节

### 2.1 核心思想
- 将检测问题转化为**主动视觉挑战-响应**：引入视觉蜜罐（adversarial honeypots）和伪装纹理，这些信号对自瞄模型是显著的，但对人类玩家不可感知或不影响。
- 将开销转移到**离线3D对抗纹理合成阶段**，运行时仅加载静态纹理，无需每帧扰动。

### 2.2 两种对抗纹理机制
- **Adversarial Camouflage Textures (ACT)**：应用于玩家模型，抑制自瞄检测置信度，使其低于阈值（<0.4），同时对人眼保持自然。
- **Adversarial Honeypot Textures (AHT)**：嵌入游戏场景中，对自瞄呈现高置信度的“玩家”外观，诱使其锁定假目标；对人类玩家不显眼。

### 2.3 关键技术细节
- **端到端可微分渲染**：基于PyTorch3D，将纹理参数τ通过渲染器R、目标检测器f可微地连接到损失函数，通过链式求导更新纹理。
- **Expectation over Renderings (EoR)**：在纹理合成过程中，对多种渲染条件（相机距离、偏航/俯仰角度、光照、背景）进行期望优化，确保纹理在不同视角下鲁棒。
- **ACT优化**：使用ℓ∞球投影梯度下降，目标是最小化检测置信度（对抗性损失），保持纹理与原始纹理的感知相似性（SSIM）。
- **AHT优化**：目标包括：(1) 排名损失（确保最高置信度检测对应蜜罐，且与次高竞争者有较大间隔）；(2) 几何一致性损失（使检测框尺寸稳定）；(3) 正则化项。
- **安全纹理管理**：
  - 离线合成纹理池，每场比赛随机选择子集（类似ASLR随机化）。
  - 纹理仅在内存中存在（赛后销毁），防止逆向提取。
  - 通过哈希完整性校验防止客户端篡改模型/UV映射。
- **蜜罐交互检测管道**：
  - **几何门控**：基于玩家位置和视角射线与蜜罐平面的交点，筛选有效交互帧。
  - **时间聚类**：使用1D DBSCAN将连续交互帧聚合成“交互轨迹”。
  - **轨迹分类**：双向LSTM对轨迹进行分类（正常扫描 vs. 异常锁定），输入包含时间差、交叉点坐标、是否在蜜罐内等特征。
  - **玩家级决策**：聚合多个轨迹预测，当异常交互数超过阈值T时判定为作弊。

## 6. 主要结论与发现
- **ACT（伪装）**：平均置信度从62.3%降至16.0%，坠幅46.3%；防御成功率（ESR）85.1%；SSIM 81.0%。
- **AHT（蜜罐）**：Decoy Success Rate 96.9%，Uniqueness Rate 96.9%，平均置信度79.3%；SSIM 71.0%。
- **轨迹检测**：准确率98.05%，精度96.53%，召回率98.98%，F1 97.74%，AUC-ROC 99.29%。玩家级误报率极低（T≥2时FPR<7×10⁻⁴，T≥3时FPR<1.84×10⁻⁵）。

- **运行时开销**：接近零（纹理替换，无每帧计算）；离线合成时间短，可跨会话重复使用。
- **用户体验**：大多数参与者认为对抗纹理与原始纹理无明显差异，自然性高。

## 7. 优点
1. **端到端设计**：首次同时实现主动防御和可解释检测，且零运行时开销。
2. **新颖的蜜罐信号**：将检测从被动行为分类转为主动挑战-响应，抗模仿能力强。
3. **3D纹理优化**：通过可微分渲染+EoR确保纹理在不同视角/光照下鲁棒，克服了2D平面限制。
4. **实用部署**：无需修改游戏引擎或渲染流水线，仅依赖标准回放日志；纹理安全管理（随机化、内存-only）增加了攻击难度。
5. **极低误报率**：通过聚合多轨迹决策，玩家级FPR可降至接近零（实际游戏为3.49×10⁻⁸），保障公平性。

## 8. 不足与局限
1. **实验规模有限**：仅20名玩家参与真实游戏，且仅使用一种开源自瞄模型（YOLOv5系列），未评估对其他检测器（如Faster R-CNN、NanoDet）或更高级模仿人类行为自瞄（如GAN轨迹、神经肌肉辅助）的鲁棒性。
2. **地图限制**：使用自定义地图而非官方地图（因官方地图内部几何数据不可公开），但作者指出检测管道与地图无关，若开发者提供元数据即可集成。
3. **防御成功率非100%**：ACT的ESR仅85.1%，AHT的DSR为96.9%，仍存在部分场景下自瞄可成功检测玩家或未被诱骗。
4. **迁移性下降**：从YOLOv5n到YOLOv5l，DSR从100%降至58.79%，表明面对更大模型时鲁棒性下降。
5. **攻击模型假设**：假设自瞄使用YOLO系列检测器，且未考虑自瞄可能对纹理进行自适应训练（如对抗训练专攻AimTrap）。虽然随机化和内存-仅策略增加了难度，但理论上仍可能被针对性攻破。
6. **蜜罐检测模型依赖训练数据**：轨迹分类器需要大量标注的交互轨迹，可能在不同游戏或地图上需要重新收集。
