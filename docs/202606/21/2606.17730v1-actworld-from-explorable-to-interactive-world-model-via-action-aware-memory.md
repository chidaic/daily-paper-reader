## 摘要
交互式世界模型旨在模拟实时用户操作下的环境动态。然而，它们的动作词汇主要局限于导航：大多数动作对应运动（如行走、转向、环顾四周），而与场景中物体的交互（例如拿起盘子、开门或触发物理响应）要么缺失，要么仅限于游戏领域，要么被降级为提示到完整视频的场景。由此产生的世界在视觉上是可以探索的，但并非真正可操作。在这项工作中，我们提出了ActWorld，一个交互式世界模型，它扩展了先前以导航为中心的生成器，以在块自回归框架内支持中期推理中的物体交互。我们认为导航与交互之间的差距源于两个瓶颈。首先，数据瓶颈：缺乏具有精确、密集标注的人-物交互数据。其次，记忆瓶颈：现有世界模型中的近因偏倚历史压缩丢弃了因果决定后续物体状态的事件转移帧，导致动作遗忘病态。在数据方面，我们构建了一个包含10万个交互视频的数据集，每个视频通过思维链推理标注了每块的描述。在模型方面，我们引入了一种分层动作感知记忆设计，通过交互重要性路由历史压缩，并辅以一个持久记忆库，在长程推理中维护事件更新和物体身份令牌。实验表明，ActWorld在单一模型中同时支持灵活的导航和丰富的物体交互，在不牺牲视角控制的情况下，与仅导航的基线相比，显著提高了交互保真度。项目页面：https://interactwm.github.io/ActWorld

## Abstract
Interactive world models aim to simulate environment dynamics under real-time user actions. However, their action vocabulary is largely confined to navigation: most actions correspond to motion (e.g., walk, turn, look around), while interaction with objects in the scene (e.g., pick up plates, open doors, or trigger physical responses) is either absent, restricted to game domains, or relegated to prompt-to-full-video scenarios. The resulting worlds are visually explorable but not truly actionable. In this work, we present ActWorld, an interactive world model that extends prior navigation-centric generators to support mid-rollout object interaction within a chunk-autoregressive framework. We argue that the navigation-interaction gap stems from two bottlenecks. First, a data bottleneck: the lack of human-object interaction data with accurate, dense labels. Second, a memory bottleneck: recency-biased history compression in existing world models discards the event-transition frames that causally determine subsequent object states, leading to an action-forgetting pathology. On the data side, we construct a 100K interaction video dataset, each annotated with per-chunk captions via chain-of-thought reasoning. On the model side, we introduce a hierarchical action-aware memory design that routes history compression by interaction importance, complemented by a persistent memory bank that maintains event-update and object-identity tokens across long rollouts. Experiments show that ActWorld supports both flexible navigation and rich object interaction within a single model, substantially improving interaction fidelity over navigation-only baselines without sacrificing viewpoint control. Project page is available at https://interactwm.github.io/ActWorld.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有交互式世界模型的动作词汇几乎全部局限于导航（行走、转向、环视），无法支持场景中物体的交互（如开门、拾取、搅拌等）。生成的场景“可浏览但不可操作”，用户只能移动视角而不能改变物体状态。
- **根源**：论文首次系统地将这一“导航-交互鸿沟”归因于两个瓶颈：
  - **数据瓶颈**：缺乏带精确、密集标注的人-物交互视频数据，现有世界模型数据集以导航为主。
  - **记忆瓶颈**：现有世界模型采用基于时间近因的历史压缩策略，将物体状态被修改的关键帧（如瓶子被装满）压缩或丢弃，导致模型“忘记”物体状态，无法正确预测后续帧（动作遗忘病理）。
- **整体目标**：将世界模型从“可探索”拓展到“可交互”，在单个实时框架内同时支持灵活导航与丰富的物体交互。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- 设计**分层动作感知记忆**，按交互重要性而非时间顺序组织历史帧，并引入持久记忆库存储事件更新和物体身份信息，克服动作遗忘。
- 构建大规模交互视频数据集，用思维链（CoT）自动为每个视频块生成详细标注。

### 2.2 关键技术细节

#### (1) 数据生成与标注管道
- 将视频切分为33帧（约1.4秒）的块（chunk）。
- 对每个块提取5个关键帧，使用冻结的VLM（GPT-5.4）进行思维链推理，输出：交互标志、交互阶段（接近、触及、接触、操作、完成、后动作）、1–2句文字描述。
- 描述文本由UMT5编码为条件嵌入。
- 另从VIPE恢复相机姿态，构建每个像素的Plücker射线。

#### (2) 键盘/鼠标控制条件化（双分支）
- **几何分支**：将Plücker射线经FiLM模块产生每token的尺度-偏移信号，作用于当前块隐藏状态。
- **符号文本分支**：将（键盘+鼠标）的81种组合映射为自然语言模板，经UMT5编码后与块描述拼接，通过交叉注意力注入。独立dropout防止捷径学习。

#### (3) 分层动作感知记忆（核心贡献）
- **局部记忆库**：用“事件感知帧重分配（EAFR）”替代基于时间间隔的历史桶划分，按交互重要性得分将关键接触/操作帧放入精细桶，而非重要但近期的帧可能退入粗糙桶。得分包含人工先验和指数时间衰减。
- **动作条件历史放大（ACHA）**：对自注意力的历史key施加可学习的动作条件放大器，使模型对因果相关历史帧的注意力增强。
- **持久动作感知记忆库（Persistent Memory Bank）**：最多16个token，FIFO管理，但来自接触/操作阶段的token被固定。包含两类：
  - **事件token**：在阶段转换（如进入操作、完成）时触发，编码转换类型、当前阶段、动作类别和视觉池化特征。
  - **物体token**：在有交互的块中，使用冻结DINOv3提取高响应patch，经MLP投影为token，编码被触摸区域的视觉签名。支持跨长导航间隙的物体身份追踪。
- 所有新增模块在初始时保持零初始化，保证与预训练检查点逐位兼容。

#### (4) 少步蒸馏
- Stage-1：分块自回归训练，5000步，48×H100。
- Stage-2：多分辨率流匹配，降低token数，5000步。
- Stage-3：对抗性DMD蒸馏，从50步教师降至3步生成器，包含I2V ODE预热和GAN目标。

- **标注**：每个视频块有密集描述、阶段标签、交互标志、动作类别。

### 3.2 Benchmark：I-Bench

## 8. 不足与局限

- **数据偏差风险**：训练数据主要来自专有视频模型合成，可能不具备真实物理环境的多样性；动作集限定40类，未覆盖更复杂的柔性物体、流体、破坏性交互等。
- **评估局限**：
  - I-Bench为合成基准，场景和动作脚本可能偏向于方法优势，与真实人类操作场景的差异未深入探讨。
  - 未与专门面向物体交互的世界模型（如PAN[36]）进行直接对比（因其非实时）。
  - 长程物体持久性（如物体被隐藏后重新出现）的评估不够系统。
- **记忆机制限制**：持久记忆库容量仅16个token，且依赖DINOv3特征，对于高度相似物体或多实例场景可能产生混淆；无显式物体ID绑定。
- **硬件需求**：训练需48×H100（约3-4天），资源成本较高；蒸馏后的3步推理仍需约0.4秒/块（推测），高分辨率实时可能仍有挑战。
- **物理合理性**：未专门评估物理一致性（如碰撞、重力），虽然VBench运动平滑度指标很高，但生成中物体交互的物理合理性可能受限。
- **标注质量**：依赖GPT-5.4进行CoT标注，可能存在幻觉或对动作阶段分类不准确，将在训练中引入噪声。
