## 摘要
世界模型综合旨在将交互体验转化为环境动态的内部模型。现有的符号方法通常拟合观察到的转换或局部规则的混合，但不能生成一个可以独立于真实环境运行的完整可执行程序。我们提出了Mind-Studio，这是一个利用大型语言模型从状态-动作-下一状态轨迹中综合可执行的pygame风格世界模型的框架。Mind-Studio将熵选择的轨迹与一个轻量级游戏技能文件相结合，该文件包含从屏幕截图中提取的对象、动作和静态场景信息。我们通过一个K步前瞻保真度协议来评估综合质量，该协议将生成的世界模型展开与来自同一状态的Real-ALE展开进行比较。在Montezuma's Revenge游戏中，Mind-Studio将所选动作的下一状态预测从PoE-World的0.3%提高到48.7%，同时验证了8个子目标中的5个；在Alien、Assault和Skiing游戏中，Mind-Studio实现了比先前学习的前瞻源更强的分支级别保真度。

## Abstract
World-model synthesis aims to turn interaction experience into an internal model of environment dynamics. Existing symbolic approaches often fit observed transitions or mixtures of local rules, but they do not produce a complete executable program that can run independently of the real environment. We present Mind-Studio, a framework that synthesizes executable pygame-style world models from state-action-next-state trajectories using large language models. Mind-Studio combines entropy-selected traces with a lightweight game skill file containing object, action, and static scene information extracted from screenshots. We evaluate synthesis quality with a K-step lookahead fidelity protocol that compares generated world-model rollouts against Real-ALE rollouts from the same state. On Montezuma's Revenge, Mind-Studio improves chosen-action next-state prediction from 0.3% for PoE-World to 48.7% while verifying 5 of 8 subgoals; across Alien, Assault, and Skiing, it achieves stronger branch-level fidelity than prior learned lookahead sources.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有符号世界模型虽然能拟合观察到的状态转换或局部规则，但无法生成一个**独立于真实环境的完整可执行程序**。在部分可观察的 Atari 游戏中，同一可见状态可能隐藏不同的内部条件，多个机制（移动、碰撞优先级、敌人行为、重生、物品收集）常在局部区域交互，因此需要一个能够编码这些机制的**可运行程序**，而不仅仅是拟合表面转换。
- **研究动机**：如何将交互经验（状态-动作-下一状态轨迹）转化为一个可执行的、对象中心的动态程序，使其能够从不同初始状态独立运行并产生连贯的行为。
- **整体含义**：提出了 Mind-Studio 框架，通过 LLM 合成 pygame 风格的世界模型，并设计了一套前瞻保真度评估协议，旨在实现比先前方法（如 PoE-World）更高的预测准确率和任务成功率。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将世界模型建模为一个**可执行的 Python 程序**（pygame 风格），具有状态更新和渲染接口。该程序由 LLM 从精选的交互轨迹和游戏技能文件中生成。
- 采用 **K 步前瞻保真度协议（K-step lookahead）** 来评估合成质量，比较世界模型展开与真实环境（Real-ALE）展开的球员位置匹配率。

### 关键技术细节
1. **符号状态表示**：使用 OC-Atari RAM 解析器将原始像素观测转换为对象中心的符号状态 `s_t`（包含球员、敌人、物品等对象的属性，如位置、大小、状态）。
2. **熵选择轨迹（Entropy-based selection）**：从日志转换 `D` 中选取信息密度高的子集 `eT`。评分函数 `H(r_i)` 考虑对象属性变化（加权）、关系变化（前后比较）和显式游戏事件。目的是将 LLM 上下文预算集中在揭示隐藏规则和上下文依赖的转换上。
3. **压缩与静态接口**：压缩机 `E(·)` 将所选轨迹转换为精简格式：记录静态接口 `P`（持久对象槽）和每步增量 `ΔP_i`（仅变化的属性，如球员位移、生命减少、接触出现/消失）。最大化每个 token 的信息量。
4. **LLM 合成**：通过提示 `g_ϕ`（LLM 合成器）结合游戏技能文件 `S_game`（编码对象类型、合法动作、静态场景布局、游戏背景和目标描述）和压缩轨迹，输出可执行 Python 程序 `M`。合成后对保留转换进行回放，选择 NSP 最高的程序作为 `M*`。
5. **LLM-as-Planner 评估框架**：共享的 LLM 规划器（π_LLM）接收当前状态、已验证的子目标前缀、和每个候选动作的 K 步世界模型预览表 `B_t`。采用子目标回滚机制（验证后前进，否则回滚到最后一个已验证检查点）。确保不同世界模型来源（LLM-as-WM、PoE-World、Mind-Studio）在相同规划器下公平比较。

## 3. 实验设计

- **数据集/场景**：使用四个 OC-Atari 游戏，覆盖不同挑战：
  - **Montezuma's Revenge**：迷宫游戏，边界条件（平台边缘、梯子入口、绳索几何、骷髅碰撞距离），隐藏变量，随机巡逻。
  - **Assault**：底部玩家炮台，敌人左右穿插躲避，动态规避模型。
  - **Alien**：随机敌人/门重生，强调随机性。
  - **Skiing**：基于速度的球员物理，持续方向 hold 会导致过度转向。
- **基准方法**：
  - **LLM-as-WM**：直接将规划器 LLM 用于预测下一状态。
  - **PoE-World**：基于程序化专家组合的集成世界模型（Piriyakulkij et al., 2025b）。
- **评估指标**：
  - **NSP（Next-State Prediction）**：球员位置像素级精确匹配率，分为**分支 NSP**（所有候选动作平均）和**所选动作 NSP**（仅规划器提交的动作）。
  - **任务成功率 SR**：Montezuma 验证的子目标比例；其他游戏使用特定成就计数（如击杀数、门数）。
- **实验设置**：
  - 每个游戏分别进行离线 NSP（固定保留状态）和在线 NSP（规划过程中实际访问的状态）。
  - 使用 Qwen3-Coder-480B 作为默认合成器和规划器；还进行了跨规划器后端的稳定性扫描（DeepSeek-V4-Flash, Llama-4-Scout-17B, Claude-4-Sonnet, Gemini-2.5-Flash）。
  - K 步前瞻：K=16（Skiing）或 K=8（其他游戏），基于最大化保留 NSP 确定。
- **消融实验**：在 Montezuma 上对合成输入进行消融：去除压缩（使用完整转换）、去除技能文件、替换为随机数据。

## 6. 主要结论与发现

- **主要结论**：Mind-Studio 能够通过单次未细化的 LLM 传递（熵选择符号轨迹 + 游戏技能文件）合成出可执行世界模型，该模型在部分可观察设置下显著优于先前的程序化世界模型（PoE-World）和 LLM-as-WM 基线。
- **具体发现**：
  - 在 **Montezuma's Revenge** 上，所选动作 NSP 从 PoE-World 的 0.3% 提升至 48.7%，验证了 8 个子目标中的 5 个。
  - 在 **Assault** 上，Mind-Studio 在所有 5 个规划器下均获得最高分支 NSP（均值 0.91 对比 PoE-World 0.36），任务结果在 3/5 规划器下最佳或持平。
  - 在 **Alien** 上，分支 NSP 略高于 PoE-World（0.34 vs 0.28），但任务结果波动较大（依赖规划器后端的路线发现能力）。
  - 在 **Skiing** 上，尽管离线 NSP 较高，但在线分支 NSP 只微弱领先（0.27 vs 0.24），任务结果多数为零，原因是 K 步 hold 协议无法捕捉持续方向的累积成本（过度转向导致速度接近零）。
  - **NSP 与任务成功关系**：在 56 个学习的来源-规划器对中，44 个案例中更高的所选动作 NSP 对应更好或持平的任务结果，证实 NSP 是有效的代理指标。
  - **消融结果**：去除压缩会显著降低所有合成器的保留 NSP；去除技能文件对 Qwen 模型伤害大；使用随机数据对 Qwen3-Coder-480B 有益但对 DeepSeek 有害，表明数据选择的重要性。

## 7. 优点

- **可执行性**：生成的世界模型是独立的 Python 程序，可运行并渲染预测状态，直接支持规划器进行前瞻模拟。
- **高效合成**：单次 LLM 传递，无需微调或大规模搜索（相比 PoE-World 的专家集成或 GIF-MCTS 的离线搜索），墙钟时间短（每游戏 < 20 分钟）。
- **新颖的保真度评估协议**：K 步前瞻协议通过比较世界模型与 Real-ALE 的像素级球员位置，提供精确的保真度测量；分离了世界模型质量与规划器实现的影响。
- **游戏技能文件设计**：可仅从像素级场景提取填充（即使无 RAM 状态也可用），缩小视觉观察与结构化动态知识之间的鸿沟。
- **对象中心程序表示**：显式优先级排序的代码控制流，编码隐藏规则（如接触优先级、重生条件），比学习权重集成更透明。
- **跨规划器后端的稳定性分析**：展示了方法在不同 LLM 下的鲁棒性，并揭示了规划器依赖的任务差异。

## 8. 不足与局限

- **实验范围有限**：仅测试了 4 个 Atari 2600 游戏，对象类型 ≤20，动作空间 ≤8；未在更丰富的词汇或连续控制场景验证。
- **确定性限制**：合成世界模型是**确定性的**，无法建模随机重生或敌人随机运动，导致预览与真实环境可能偏差。
- **动作-保真度不平衡**：某些情况下（如 Montezuma）LLM-as-WM 在静止动作上保真度高而 Mind-Studio 在移动动作上保真度高，但聚合分支 NSP 可能掩盖这种差异。
- **几何敏感错误**：误差集中在复合交互（绳索/骷髅、梯子-平台重叠），需要针对稀有交互类型的定向轨迹收集。
- **K 步 hold 协议局限**：在 Skiing 中，hold 固定动作 K 帧无法反映累积方向成本，导致在线 NSP 与任务成功解耦。
- **任务成功依赖探索质量**：Alien 中尽管 NSP 较强，但任务结果因规划器后端的路线发现能力而异，表明世界模型质量并非唯一瓶颈。
- **无细化和验证循环**：可选细化循环在实验中效果不一致，最终采用单轮合成，可能导致遗留部分隐藏规则。
