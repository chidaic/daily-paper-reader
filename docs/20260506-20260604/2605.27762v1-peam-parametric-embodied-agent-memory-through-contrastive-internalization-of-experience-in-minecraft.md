## 摘要
我们提出了PEAM，一种在《我的世界》中的参数化具身智能体记忆框架，它将智能体记忆从推理时的检索转变为通过经验内化的参数驻留技能。PEAM将一个用于开放推理的慢速审慎大语言模型与一个用于反射性执行已整合技能的快速参数化模块配对使用。快速模块采用多模态专家混合LoRA架构，具有按类别物理隔离的适配器，实现了无灾难性遗忘的持续参数级学习。我们将失败视为首要训练信号：通过联合行为克隆和对比目标将失败-修正轨迹对内化，使智能体不仅能学到成功的方法，还能了解修正动作与失败动作的不同。为了控制整合，PEAM引入了一个用于决定哪些经验应该内化的参数化价值分数，以及一个无标度自触发整合机制，用于决定何时内化而无需任务特定的手动调整阈值，随着触发器跨任务分布进行无重调的迁移，使智能体自我进化。在《我的世界》中的实验表明，PEAM提高了长时任务性能，减轻了对先前整合技能的遗忘，并在参数化与检索效率方面优于基于检索的具身智能体和参数化记忆变体。

## Abstract
We present PEAM, a Parametric Embodied Agent Memory framework in Minecraft that transforms agent memory from inference-time retrieval into parameter-resident skills internalized through experience. PEAM pairs a slow deliberative LLM for open-ended reasoning with a fast parametric module for reflexive execution of consolidated skills. The fast module is a multimodal Mixture-of-Experts LoRA architecture with per-category physically isolated adapters, enabling parameter-level continual learning without catastrophic forgetting. We treat failure as a first-class training signal: failure--correction trajectory pairs are internalized through a joint behavioral-cloning and contrastive objective, so the agent learns not only what succeeds but also how corrected actions differ from failed ones. To govern consolidation, PEAM introduces a parameterization-worthiness score for deciding which experience should be internalized, and a scale-free self-triggered consolidation mechanism for deciding when to internalize without task-specific hand-tuned thresholds, making the agent self-evolving as the trigger transfers across task distributions without re-tuning. Experiments in Minecraft show that PEAM improves long-horizon task performance, mitigates forgetting on previously consolidated skills, and improves parametric-versus-retrieval efficiency over retrieval-based embodied agents and parametric memory variants.

### 1. 论文的核心问题与整体含义（研究动机和背景）

当前基于大语言模型（LLM）的具身智能体普遍采用非参数化记忆（如检索式技能库、反思日志），即所有经验在推理时被重新注入上下文，而智能体的**参数化策略模型不变**。这种设计带来三个实际代价：每次调用都消耗上下文预算（检索+提示构建增加延迟）、经验必须重复重新引入才能被利用、以及**未能系统性地将外部积累的经验固化为模型参数能力**。受认知科学中“互补学习系统”理论启发（快速稀疏的情景记忆→慢速分布式的参数记忆，通过离线整合完成），论文提出**参数化具身智能体记忆框架（PEAM）**，旨在将智能体的经验从“推理时检索”转变为“参数驻留技能”，实现持续学习并缓解灾难性遗忘。

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
PEAM 采用**双层级架构**：
- **慢速审慎层（π_slow）**：使用 LLM（如 GPT-4o）处理开放推理、代码生成、结果验证。
- **快速参数化层（π_fast）**：基于 Qwen3-VL-8B-Instruct 的多模态 Mixture-of-Experts LoRA 架构，每个技能类别拥有物理隔离的适配器（Adapter），实现低延迟反射式执行已整合技能。

两个层级通过一个**整合管道（Consolidation Pipeline）**耦合，该管道包含三个决策机制：
1. **“什么”内化**：参数化价值评分（Parameterization-Worthiness, PV）——对候选技能从成本、稳定性、冗余性、干扰性四个维度评分。
2. **“何时”内化**：无标度自触发整合（Self-Triggered Consolidation, STC）——基于技能各自的失败率与历史基线的统计检验，无需任务特定阈值。
3. **“如何”内化**：联合目标——行为克隆（BC）+ 直接偏好优化（DPO），在成功轨迹和失败-修正轨迹对上训练对应的适配器。

#### 关键技术细节
- **轨迹存储**：经验池 E 包含成功轨迹 D_succ 和失败-修正对 D_cpair = {(x, τ_f, τ_c, c)}，其中 τ_f 失败，τ_c 在相同上下文 x 下成功，c 为技能类别。
- **整合目标**：  
  L_PEAM(θ_c) = L_BC(θ_c; D_succ^{(c)}) + λ L_DPO(θ_c; D_cpair^{(c)})  
  DPO 项使适配器偏好修正后的轨迹，BC 项提供绝对模仿信号（格式脚手架），避免 DPO 单独使用时生成的代码无法被解析。
- **参数化价值评分**：PV(s) = w1·U_cost + w2·U_stab - w3·P_redund - w4·R_forget，各分量基于未来调用频率估计、上下文成功率、代码相似度、类别重复性。
- **自触发整合条件**：Z(s) > z_α 且 PV(s) 在 top-q 分位内，其中 Z(s) 基于最近 W 次执行失败率与历史基线 B 的比率检验。
- **隔离机制**：每种类别的 LoRA 适配器物理隔离，更新时只影响该类别，从根本上阻断跨类别遗忘。

#### 算法流程（文字说明）
Algorithm 1 描述了 PEAM 的一次执行与整合循环：  
1. 根据任务 t 路由到对应技能类别 c。  
2. 若已有整合技能，使用 π_fast 生成代码并执行；若验证失败，回退至 π_slow。若无技能，直接使用 π_slow。  
3. 将轨迹及结果存入经验池 E。  
4. 从 E 提取候选技能，对满足 PV 阈值和 STC 触发条件的候选，仅更新其所属类别的适配器。

### 6. 论文的主要结论与发现

1. **任务性能提升**：PEAM 成功率达到 69.7%，比 VOYAGER（54.5%）高 15.2 个百分点（p=0.018），优于最强检索基线 Optimus-1-rep（60.6%）。
2. **效率优势**：中位调用延迟降低 42%（5.5s→3.2s），每任务 tokens 减少 85%（31.2K→4.6K）。
3. **遗忘抵抗**：在顺序整合后，PEAM 对最早 craft 技能的保留率为 100%，而单共享 LoRA 下降 32.4%，EWC 下降 43.3%，全微调下降 78.5%。
4. **选择性整合有效**：PV 评分与简单启发式（成功率≥0.8且检索次数≥15）的 Spearman 相关系数为 -0.375，说明 PV 选择不同的技能；STC 自触发在两种分布下产生可解释的触发模式，而固定阈值需要重新调整。
5. **BC 不可或缺**：纯 DPO 的适配器无法生成解析器可接受的代码（0/12），联合 BC+DPO 达到 12/12，验证了 BC 的作用。
6. **额外方法论发现**：
   - 前向偏好评价（margin）不能预测生成路径的可部署性。
   - 量化部署存在零化低幅更新、延迟过高、解析器 token 预算冲突等问题。
   - 失败类型影响 cpair 产量：导航类失败多因环境限制，不适合对比性参数整合。
   - 轨迹重切片可作为跨分布评估的替代方案。

### 7. 优点：方法或实验设计上的亮点

### 8. 不足与局限

- **单一环境**：所有实验仅在 Minecraft 中进行，无法直接推广至机器人操作、网页智能体等其他具身领域。转移性未验证。
- **类别覆盖有限**：仅整合了 craft、gather、combat 三类技能，其余行为（如导航）由慢速层处理。适配器数量随类别线性增长，扩展性需进一步评估。
- **动作语法固定**：经验内化依赖特定的 JavaScript 动作语法，BC 拯救格式的作用在其他动作空间（如 API 调用）中是否成立未说明。
- **PV 权重固定**：四个权重通过网格搜索选定后固定，非自适应。若任务分布变化，最优权重可能偏移。
- **跨分布评估方法**：STC 的鲁棒性通过“轨迹重切片”模拟跨分布，而非真实的独立收集分布，结论有一定合成性。
- **伦理风险**：将经验内化为参数后，模型变得更难审计；若验证环节出现错误，错误行为可能被强化。论文提及了审计工具和安全的必要性，但未实现。
- **硬件依赖性**：快速模块需要 8 张 A100 在 bf16 下服务，消费级设备（如 RTX 4070）无法直接部署，限制了实际落地。
- **未探索其他对比学习框架**：仅使用 BC+DPO，未与 KTO 或其他偏好对齐方法比较。
