## 摘要
在开放世界游戏中，长时域多模态智能体必须在严格的令牌和延迟预算下，在众多低级交互中保持目标导向。现有方法通常在代价高昂的逐步骤推理与反应式执行之间权衡，后者可能导致漂移、重复失败以及恢复不佳。我们的关键思想是在局部稳定片段中复用策略推理，并在事件边界处重新调用。我们提出SPIKE，一种面向低成本长时域游戏控制的自适应双控制器框架。其策略控制器执行低频全局规划、失败分析和恢复，而反应控制器在严格令牌预算下处理快速局部执行。事件触发器监控视觉变化、任务进度、重复动作和失败信号，以决定控制应保持反应式还是升级到策略推理。分层记忆将状态-动作记忆库（SA-MB）中的短期经验复用与状态动作知识图谱（SA-KG）中的结构化证据分离，使每个控制器能检索所需上下文。该设计在多个反应步骤中复用策略提议，当计划过时时支持局部覆盖，并将昂贵的推理保留在额外思考有用的时刻。在StarDojo的Lite-100分割上，SPIKE相比最强Lite-100基线将Lite-100成功率（SR）提升5.0个百分点（相对提升38.5%），相比最强预算基线将预算成功率提升9.3个百分点（相对提升75.6%）。同时令牌消耗降低54.9%，延迟降低40.8%。消融实验表明，事件触发、反应覆盖和异构记忆均有助于成功和恢复，支持选择性推理而非每一步推理。

## Abstract
Long-horizon multimodal agents in open-world games must stay goal-directed across many low-level interactions under tight token and latency budgets. Existing approaches often trade off costly per-step reasoning against reactive execution that can drift, repeat failures, and recover poorly. Our key idea is to reuse strategic reasoning across locally stable segments and reinvoke it at event boundaries. We present SPIKE, an adaptive dual controller framework for cost-efficient long-horizon game control. Its Strategic Controller performs low-frequency global planning, failure analysis, and recovery, while its Reactive Controller handles fast local execution under a strict token budget. An Event Trigger monitors visual change, task progress, repeated actions, and failure signals to decide when control should stay reactive or escalate to strategic reasoning. Hierarchical Memory separates short-term experience reuse in the State-Action Memory Bank (SA-MB) from structured evidence in the State Action Knowledge Graph (SA-KG), allowing each controller to retrieve the context it needs. This design reuses strategic proposals over multiple reactive steps, supports local override when plans become stale, and reserves expensive reasoning for moments where extra deliberation is useful. On the Lite-100 split of StarDojo, SPIKE improves Lite-100 success rate (SR) by 5.0 percentage points (38.5% relative) over the strongest Lite-100 baseline and Budgeted SR by 9.3 points (75.6% relative) over the strongest budgeted baseline. It also reduces token consumption by 54.9% and latency by 40.8%. Ablations show that event triggering, reactive override, and heterogeneous memory each contribute to success and recovery, supporting selective reasoning rather than reasoning at every step.

### 1. 核心问题与整体含义

- **核心问题**：现有方法面临“规划-延迟-记忆”三难困境。
  - **重度规划型**（如CRADLE）：每一步都进行深度推理，效果好但成本极其高昂（令牌消耗大、延迟高）。
  - **轻量反应型**（如ReAct）：执行速度快、成本低，但容易发生任务漂移、重复失败，且恢复能力差。
- **整体含义**：SPIKE的核心思想是**将策略推理视为一种稀缺资源，只在关键事件节点上（如场景突变、进度停滞、重复失败）调用昂贵的策略推理，而在稳定的局部执行阶段复用已有的策略规划**。这种方法旨在打破成本与性能的权衡，实现既高效又有弹性的长时域智能体控制。

### 2. 方法论

- **核心思想**：基于双过程认知理论，构建一个**事件触发、自适应切换**的双控制器框架。将昂贵的“慢思考”与便宜的“快思考”分离，并引入控制器专属的分层记忆系统。
- **关键技术细节**：
  - **事件触发（Event Trigger）**：作为在线代理，决定何时需要进行策略推理。它监控以下信号，当任一信号超过阈值时触发策略控制器（`gt=1`）：
    - **视觉变化（`dt`）**：连续两帧图像之间的余弦距离过大，表示场景突变。
    - **零进度停滞（`zt`）**：任务进度计数器长时间无变化。
    - **行为重复（`rt`）**：在最近`W`步中检测到重复动作循环。
    - **执行失败（`lt`）**：检测到执行错误。
    - **周期刷新（`ct`）**：为避免长期不进行顶层检查，设置了最大反应步数`T`。
  - **自适应双控制器**：
    - **策略控制器（Strategic Controller）**：在事件触发时**低频**运行。负责信息收集、任务推理、失败分析和生成未来`T`步的短期动作规划。能力强大但成本高。
    - **反应控制器（Reactive Controller）**：在非触发时**高频**运行。在严格的令牌预算下，基于策略控制器的最新规划和当前观察执行局部动作。它只能进行有限的本地修正（如调整面向、避障），不能更改子目标。如果无法推进，会请求事件触发。
  - **分层记忆（Hierarchical Memory）**：
    - **全局记忆（Global Memory）**：维护一个固定大小的近期交互窗口，用于保证任务的连续性。
    - **状态-动作记忆库（SA-MB）**：一个**低延迟的局部动作先验库**。为反应控制器提供快节奏的本地提示，强调近期、高回报和成功经验的重用。
    - **状态动作知识图谱（SA-KG）**：一个**结构化策略证据图**。为策略控制器提供规划校验和重新规划的证据。存储了已验证的状态转移关系（状态节点-动作边），通过向量搜索和图遍历相结合的方式检索。
- **流程**：在每个步骤，智能体获取观察，事件触发器决定使用哪个控制器。若使用策略控制器，则检索SA-KG证据，进行推理并生成动作规划。若使用反应控制器，则检索SA-MB提示，根据当前规划执行动作。执行后，更新记忆库和全局历史。

### 6. 主要结论与发现

- **主要结论**：将结构性控制与结构性记忆相结合，对于长时域游戏智能体至关重要。

  - **预算效率高**：在预算成功率上，SPIKE比最强基线（StarDojo）高出9.3个百分点，证明了其在有限推理资源下的优越性。
  - **恢复能力强**：SPIKE的恢复/卡住比是所有方法中最高的。
  - **组件贡献**：消融实验证明，事件触发、反应覆盖、SA-MB和SA-KG都是互补且不可或缺的组成部分。

### 7. 优点

- **方法创新**：
  - 提出了一个**优雅且实用的方法论**，将资源分配问题与智能体控制问题结合，对长时域任务具有普适性意义。
  - **事件触发机制**设计精巧，利用多模态信号作为决策依据，无需依赖任务模型或真实奖励。
  - **分层记忆**是亮点，将短期局部经验（SA-MB）和长期结构化知识（SA-KG）分离，减少了检索噪音，并为不同控制器提供了最合适的上下文。
- **实验严谨**：
  - 实验设计层次分明，从整体性能到组件贡献再到超参数敏感性，逻辑清晰。
  - 考虑了**成本效率**（令牌、延迟）和**恢复能力**，而不仅仅是最终成功率，评估维度全面。
  - 跨基准（RDR2）的验证增强了结论的普适性。

### 8. 不足与局限

- **对视觉变化和任务信号的依赖**：事件触发依赖于可量化的视觉变化和任务进度信号。当游戏状态变化细微，或视觉变化不发生时，可能无法正确触发策略推理，导致误触发或漏触发。
- **记忆污染风险**：SA-MB和SA-KG虽然经过过滤，但长期运行下仍有存储错误或陈旧知识的风险，可能导致“习惯性错误”。
- **事件阈值需要校准**：事件触发的固定阈值在StarDojo上表现良好，但迁移到动作粒度、视觉风格和反馈机制完全不同的新游戏时，需要手动或自动重新校准。
- **任务类型表现不均**：在战斗和社交类任务上，SPIKE的成功率虽然高于基线但仍显著偏低（分别为8.3%和10.7%），说明在处理动态敌人和长距离导航交互时仍面临挑战。
- **计算延迟**：尽管延迟降低了40.8%，但43.5秒/步的延迟对于实时游戏场景仍然过高，部署于毫秒级交互的游戏中还有很大距离。
